# 多模态大模型幻觉检测与评估实验报告

## 摘要

本报告围绕多模态大模型（Multimodal Large Language Models, MLLMs）在视觉问答与图文推理任务中的幻觉现象展开研究，重点关注幻觉定义、自动化检测方法、模型间差异、CoT（Chain-of-Thought）对幻觉暴露的影响，以及自动检测结果与人类判断之间的一致性。实验选择 POPE 作为通用视觉问答任务，选择 MathVista 作为图文数学推理任务，并比较 Gemini 与 Qwen 两类多模态模型在 Direct answer 与 evidence-grounded CoT 两种 prompt 设置下的表现。

本实验将多模态幻觉定义为：模型回答中出现了无法被图像证据、题目约束、选项、参考答案、数学或图形规则、以及回答内部逻辑支持的可检查断言。基于该定义，报告采用 `OBJ/ATT/SPA/IR/CI/INC/SO` 七类细粒度标签，将幻觉划分为视觉事实幻觉、推理幻觉、内部一致性幻觉和语义过度推断。自动检测方面，实验先构建 baseline zero-shot judge，再进一步使用 Zero-shot Judge v2。v2 的核心不是训练新模型，而是通过 claim-level evidence checking、evidence source 标注、support status 判断和 deterministic aggregation，使自动判定结果更可解释、更稳定。

实验结果显示，POPE Direct 设置下幻觉主要集中在对象存在性错误（`OBJ`），而 MathVista 中主要错误集中在无效推理（`IR`）和属性/数值读取错误（`ATT`）。CoT 设置下，所有任务和模型组合的 response-level hallucination rate 均上升，但这不能简单解释为 CoT 使模型更容易幻觉，因为 CoT 同时显著增加了每条回答中可检查 claim 的数量。Human-as-Judge 部分使用 40 条人工标注样本进行验证，并在复核后将 `ann_0020` 从原始 `INC` 幻觉标注修正为 non-hallucination。按修正版人工标签计算，Zero-shot Judge v2 取得 Accuracy=0.8250、F1=0.7200、Cohen's Kappa=0.5930、MCC=0.5940，相比 baseline 减少了 false positives，并更接近人类判断边界。最后，报告进一步讨论了跨任务 claim-level judge 的适用性，并将方法扩展到 XLRS-bench 遥感场景，作为自动诊断性质的加分实验。

**关键词：** 多模态大模型；幻觉检测；视觉问答；MathVista；POPE；Zero-shot Judge；Human-as-Judge；Chain-of-Thought

## 1. 引言

多模态大模型能够同时处理图像与文本输入，并在视觉问答、图表理解、数学推理、遥感图像分析等任务中表现出较强的泛化能力。然而，模型生成的回答并不总是可靠。即使回答语言流畅、形式完整，也可能包含图像中不存在的对象、错误的视觉属性、未经支持的推理步骤，或者与最终答案相矛盾的中间解释。这类现象通常被称为多模态幻觉。

在多模态任务中，仅用最终答案是否正确来评估模型是不充分的。一个模型可能给出正确 final answer，但在 reasoning 中编造不存在的视觉证据；也可能 final answer 错误，但错误来源是格式不匹配或题目理解偏差，而不一定构成幻觉。因此，本实验的核心思路是将判断单位从 response-level final answer 推进到 claim-level evidence checking：检查模型回答中的每个可验证 claim 是否被图像、题干、选项、参考答案、数学规则、图形规则或回答内部逻辑支持。

本报告围绕以下研究问题展开：

1. **RQ1：不同任务中的幻觉类型是否不同？**  
   POPE 主要测试对象存在性判断，预期以 `OBJ` 为主；MathVista 涉及视觉属性读取、图表理解和数学推理，预期 `ATT` 与 `IR` 更突出。

2. **RQ2：CoT 是否会改变幻觉率和幻觉类型分布？**  
   CoT 可能帮助模型显式组织视觉证据和推理链，但也会暴露或生成更多可检查 claim。

3. **RQ3：不同模型的 hallucination profile 是否不同？**  
   本实验比较 Gemini 与 Qwen 在相同任务和 prompt 设置下的幻觉率与标签分布。

4. **RQ4：自动 Judge 与人类判断是否一致？**  
   本实验使用 40 条人工标注样本作为 Human-as-Judge gold label，验证自动检测方法与人类判断的一致性。

本报告的主要贡献包括：

- 构建了适用于多模态回答的幻觉定义与 `OBJ/ATT/SPA/IR/CI/INC/SO` 分类体系；
- 实现并比较了 baseline zero-shot judge 与 Zero-shot Judge v2；
- 在 POPE 与 MathVista 上完成 Gemini 与 Qwen 的 Direct/CoT 对照实验；
- 使用 40 条人工标注样本评估自动 Judge 与 Human-as-Judge 的一致性；
- 分析 CoT 对 claim exposure 和 response-level hallucination rate 的影响；
- 总结典型 disagreement 与 failure cases；
- 将统一 claim-level judge 扩展到跨任务分析和 XLRS-bench 遥感场景。

## 2. 幻觉定义与分类体系

### 2.1 多模态幻觉定义

本实验将多模态大模型幻觉定义为：

> 模型回答中出现了与图像、题干、选项、参考答案、数学或图形规则、回答内部逻辑不一致，或者缺乏足够证据支持的可检查断言。

该定义强调三个要点：

1. **判断对象是 claim，不只是 final answer。**  
   正确答案仍可能包含幻觉推理；错误答案也不一定自动等于幻觉。

2. **证据来源必须明确。**  
   每个 claim 应判断它应该依据 image、question、choices、reference answer、math rule、diagram rule 还是 internal consistency。

3. **response-level 结果应由 claim-level 检查聚合得到。**  
   本实验不直接信任模型 judge 的顶层总结，而是基于结构化 claim checks 程序化推导最终标签。

### 2.2 幻觉分类体系

本实验采用四类粗粒度幻觉，并进一步映射到七个细粒度标签。

| 粗粒度类型 | 细粒度标签 | 核心判断依据 |
|---|---|---|
| 视觉事实幻觉 | `OBJ`, `ATT`, `SPA` | 对象、属性、数量、空间关系是否被图像支持 |
| 推理幻觉 | `IR`, `CI` | 推理步骤、数学规则、上下文约束是否成立 |
| 内部一致性幻觉 | `INC` | visual evidence、reasoning、final answer 是否前后一致 |
| 语义过度推断 | `SO` | 是否从图像过度外推身份、意图、年代、功能或场景语义 |

### 2.3 视觉事实幻觉

视觉事实幻觉指模型对图像中可直接验证的视觉事实作出错误或无依据陈述，主要包括对象、属性和空间关系。

#### OBJ：对象或类别错误

`OBJ` 指模型错误判断图像中某个对象是否存在，或将对象类别识别错误。

示例：

1. POPE 问题询问图中是否有 handbag。如果图中没有 handbag，但模型回答 yes，则属于 object presence hallucination。
2. 图像中只有狗和人，模型却声称图中有 cat，属于不存在对象的虚构。

#### ATT：属性、数量或数值错误

`ATT` 指模型错误描述对象属性，包括颜色、数量、大小、形状、状态、OCR 文本、图表值或数学图形中的视觉数值。

示例：

1. MathVista 中，模型声称陆地上有 16 只动物、水中有 5 只动物，但人工检查认为数量读取错误，属于视觉计数错误。
2. 图表中 Firebrick 与 Midnight Blue 数值相等或接近，但模型断言其中一个明显更大，属于图表属性或数值读取错误。

#### SPA：空间关系错误

`SPA` 指模型错误描述对象位置、方向、相邻关系、包含关系或整体空间布局。

示例：

1. 图像中物体 A 位于物体 B 右侧，但模型声称 A 在 B 左侧。
2. 遥感图像中，模型错误描述道路、水体、建筑群之间的相邻关系，导致空间布局判断不成立。

### 2.4 推理幻觉

推理幻觉指模型使用了不被题目条件、视觉证据、数学规则或图形规则支持的中间推理。

#### IR：无效推理

`IR` 指模型推理步骤不成立，使用错误公式、错误条件、错误计算，或从证据到结论的推导不被支持。

示例：

1. 几何题中，模型在没有足够对应边或对应角条件的情况下，直接推出两条线段相等，并据此计算答案。
2. MathVista 中，模型给出面积、斜率或比例结论，但中间公式与图中数值不匹配，或缺少必要计算依据。

#### CI：上下文不一致

`CI` 指回答与题目约束、选项、reference answer context 或要求的答案格式冲突。

示例：

1. 题目要求从 A/B/C/D 中选择，但模型输出一个不在选项中的自由文本答案。
2. 问题要求比较两个图表值，模型却回答了无关对象或无关数值。

### 2.5 内部一致性幻觉

`INC` 指模型回答内部前后矛盾，或 reasoning、visual evidence 与 final answer 在同一题干语义框架下无法同时成立。

示例：

1. 模型前文说图中没有目标对象，后文却用该对象作为证据回答 yes，属于同一回答内部自相矛盾。
2. 模型 reasoning 明确推出选项 A，但 final answer 输出选项 B，且没有新的证据或修正说明，属于推理链与最终答案不一致。

需要注意的是，`INC` 不能只根据局部措辞判断。对于比较题、填空题和 yes/no 题，必须先确认题干主语、比较方向和答案格式，再判断 reasoning 与 final answer 是否真的矛盾。

### 2.6 语义过度推断

`SO` 指模型从图像可见信息过度外推到身份、意图、年代、用途、社会语义或不可见背景知识，且缺少足够证据支持。

示例：

1. 问题问图中有多少人出生在二战结束之后，模型直接回答 `0`。普通图像通常无法直接证明人物出生年代，因此该回答隐含了缺乏证据的语义推断。
2. 遥感图像中，模型仅凭建筑外观断言该区域是军事基地或工业污染源，如果缺少明确地物、文字或上下文证据，就属于语义过度推断。

## 4. 自动幻觉检测方法

### 4.1 Baseline Zero-shot Judge

Baseline detector 是一个 prompt-based MLLM-as-a-Judge 方法，不训练额外模型，而是直接调用多模态 LLM 作为 hallucination judge。每条 model response 会与对应的图像、问题、选项、参考答案、解析出的 visual evidence、reasoning 和 final answer 一起输入 judge prompt。

Baseline 的基本流程为：

```text
理解问题要求
→ 从 model response 中抽取 explicit verifiable claims
→ 判断每个 claim 是 supported / contradicted / unverifiable
→ 将问题 claim 映射到 OBJ / ATT / SPA / IR / CI / INC / SO
→ 输出 response-level is_hallucination 与 claim_checks
```

Baseline 的优点是实现简单、无需训练、可解释性较强，并且可以快速得到 response-level 与 claim-level 判断。但它存在以下局限：

1. **缺少显式 evidence_source。**  
   Judge 不一定清楚每个 claim 应该依据图像、题干、选项、reference answer、数学规则还是内部一致性判断。

2. **缺少 non_claim / not_applicable 机制。**  
   格式文本、短答案、answer token 或不可检查内容可能被误当成可疑 claim。

3. **容易过度惩罚 unverifiable claim。**  
   在 reference answer 不可用、答案较短或题目本身允许估计时，baseline 容易产生 false positive。

4. **顶层 summary 与 claim-level 判断可能不一致。**  
   Raw judge 输出的 `is_hallucination` 不一定与 claim checks 中的 evidence 判断完全一致。

因此，baseline 在 Human-as-Judge 上表现为“高召回、低精度”：能发现较多人工标注的 hallucination，但误报也较多。

### 4.2 Zero-shot Judge v2

Zero-shot Judge v2 的核心改进是将检测过程强化为 taxonomy-constrained claim-level evidence checking。v2 仍然不训练模型，也不是监督分类器；它仍属于 prompt-based zero-shot judge，但通过更严格的输出结构和程序化归一化来提高稳定性。

v2 的检测流程为：

```text
固定幻觉定义与标签体系
→ 从完整 model response / CoT 中抽取 atomic checkable claims
→ 为每个 claim 标注 evidence_source
→ 判断 support_status
→ 将问题 claim 映射到 OBJ / ATT / SPA / IR / CI / INC / SO
→ 单独判断 final_answer correctness
→ 只基于 normalized claim_checks 聚合 response-level detector 结果
```

v2 的关键字段包括：

| 字段 | 含义 |
|---|---|
| `claim_checks` | claim-level 判断来源，是 response-level 聚合的核心依据 |
| `claim_type` | claim 类型，如 object、attribute、spatial、reasoning、causal、inconsistency、semantic、answer、non_claim |
| `evidence_source` | 判断 claim 应依据的证据来源，如 image、question、choices、reference_answer、math_rule、diagram_rule、internal_consistency、none |
| `support_status` | claim 支持状态，包括 supported、contradicted、unverifiable、not_applicable |
| `fine_label` | 细粒度幻觉标签，包括 OBJ、ATT、SPA、IR、CI、INC、SO、None、Unclear |
| `summary_consistent_with_claims` | raw judge summary 是否与 claim-level 判断一致 |
| `raw_normalized_mismatch` | raw 输出与程序化归一化结果是否不一致 |

v2 的一个关键设计是：`hallucination_labels` 不再要求 judge 直接输出，而是由程序根据 `claim_checks[].support_status` 与 `claim_checks[].fine_label` 确定性推导。这减少了 LLM 输出不稳定性，也避免了顶层标签与 claim-level evidence 不一致的问题。

### 4.3 Baseline 与 v2 对比

| 维度 | Baseline Judge / v1 | Zero-shot Judge v2 |
|---|---|---|
| 方法性质 | Prompt-based zero-shot GPT judge | Prompt-based zero-shot GPT judge |
| 是否训练模型 | 否 | 否 |
| 判断粒度 | response-level 判断为主，同时输出 claim checks | taxonomy-constrained claim-level evidence checking |
| Evidence source | 不显式 | 每个 claim 必须标注 `evidence_source` |
| Claim 状态 | supported / contradicted / unverifiable | supported / contradicted / unverifiable / not_applicable |
| Non-claim 处理 | 无显式类型 | 增加 `non_claim` 与 `not_applicable` |
| Final answer 处理 | 容易影响 hallucination 判断 | answer correctness 与 hallucination 判断分离 |
| Reference unavailable | 容易导致 answer-only mismatch 误判 | 明确不能作为 contradiction evidence |
| 聚合方式 | 较依赖 judge 顶层 summary | 只基于 normalized `claim_checks` 聚合 |
| 主要倾向 | 高 recall、低 precision | false positives 减少，更接近人工判断边界 |

### 4.4 方法定位

Zero-shot Judge v2 的价值不在于“完全替代人工标注”，而在于提供一个可解释、可批量运行、可与人工判断校准的诊断工具。它适合用来发现不同任务、模型、prompt 下的 hallucination pattern，但仍需要 Human-as-Judge 结果来校准其判断边界。

## 5. 评估指标

本实验使用 response-level、human-alignment、claim-level 三类指标。

### 5.1 综合指标说明

本实验将指标分为三组，并用两张总表展示：第一张表合并 response-level 与 Human-as-Judge 对齐结果，第二张表单独展示 claim-level 诊断结果。

- **Response-level 指标**：`Hallucination Rate = hallucination_count / total_responses`，表示某一任务、模型、prompt 和 judge 组合中被自动 detector 判为 hallucination 的回答比例。
- **Human-as-Judge 对齐指标**：将 40 条人工标注样本作为 gold label，计算自动 detector 与人工判断之间的一致性，包括 Precision、Recall、F1、Accuracy、Cohen's Kappa 和 MCC。
- **Claim-level 指标**：基于 Zero-shot Judge v2 归一化后的 `claim_checks`，统计每条回答平均 claim 数、可检查 claim 数、unsupported claim 数，以及 unsupported claim ratio 和 label-triggering unsupported claim ratio。

### 6.2 任务差异：POPE 与 MathVista

POPE 的 direct 设置主要出现 `OBJ` 类型幻觉。POPE Gemini Direct 中 61 个 hallucination 全部为 `OBJ`；POPE Qwen Direct 中 69 个 hallucination 也全部为 `OBJ`。这符合 POPE 的任务性质：它主要检验模型是否正确判断图像中是否存在某个目标对象。

MathVista 的幻觉结构更复杂。该任务不仅要求模型识别图像对象，还要求读取图表、几何图形、数量、长度、角度、斜率等视觉属性，并将这些视觉信息接入数学推理。因此，MathVista 的主要问题集中在：

- `IR`：无效推理、错误公式、错误计算或不成立的图形推导；
- `ATT`：数量、图表值、视觉属性或数学图形读数错误；
- `INC`：reasoning 与 final answer 不一致。

从结果看，MathVista 的 hallucination rate 整体高于 POPE。例如 Gemini Direct 在 POPE 上为 6.78%，在 MathVista 上为 18.87%；Qwen Direct 在 POPE 上为 7.67%，在 MathVista 上为 23.15%。这说明图文数学推理任务比对象存在性 VQA 更容易触发复杂证据边界问题。

### 6.3 Prompt 差异：Direct 与 CoT

在所有任务和模型组合中，CoT 设置下的 response-level hallucination rate 都高于 Direct 设置：

| Dataset | Model | Direct Hallucination Rate | CoT Hallucination Rate | Change |
|---|---|---:|---:|---:|
| POPE | Gemini | 6.78% | 21.00% | +14.22% |
| POPE | Qwen | 7.67% | 20.00% | +12.33% |
| MathVista | Gemini | 18.87% | 35.02% | +16.15% |
| MathVista | Qwen | 23.15% | 45.53% | +22.38% |

注：表中差值按展示到两位小数的 hallucination rate 计算；若使用未四舍五入的原始值，MathVista Qwen 的变化为 +22.37%。

这一结果不能简单解释为“CoT 一定让模型更差”。CoT 会让模型输出更多 visual evidence、reasoning steps 和中间结论，这会显著增加 detector 可以检查的 claim 数量。response-level 判断只要发现一个关键 claim unsupported 或 contradicted，就可能将整条回答判为 hallucination。因此，CoT 幻觉率上升既可能来自真实错误增加，也可能来自 claim exposure 增加。

### 6.4 模型差异：Gemini 与 Qwen

在 POPE 上，Gemini 与 Qwen 的 hallucination rate 接近，说明对象存在性判断对两者都是较基础的视觉 grounding 测试。POPE Direct 中，Gemini 为 6.78%，Qwen 为 7.67%；POPE CoT 中，Gemini 为 21.00%，Qwen 为 20.00%。

在 MathVista 上，模型差异更明显。Gemini Direct 为 18.87%，Qwen Direct 为 23.15%；Gemini CoT 为 35.02%，Qwen CoT 为 45.53%。这表明复杂图文数学推理会放大模型在视觉属性读取、数学推导和语言题干绑定方面的差异。

需要注意的是，这一结论仅限于当前数据、prompt 和 judge 设置。它说明两类模型在当前实验中的 hallucination profile 不同，而不应被表述为对模型总体能力的绝对排名。

## 7. CoT 对幻觉的影响：Claim Exposure 分析

### 7.1 Claim 数量变化

CoT 设置下，每条回答中的 claim 数量显著增加。

| Dataset | Model | Direct Claims / Resp. | CoT Claims / Resp. | Δ Claims / Resp. | Direct Response Hallucination Rate | CoT Response Hallucination Rate |
|---|---|---:|---:|---:|---:|---:|
| POPE | Gemini | 1.12 | 5.57 | +4.45 | 6.78% | 21.00% |
| POPE | Qwen | 1.13 | 6.60 | +5.47 | 7.67% | 20.00% |
| MathVista | Gemini | 2.50 | 8.15 | +5.66 | 18.87% | 35.02% |
| MathVista | Qwen | 1.85 | 6.86 | +5.01 | 23.15% | 45.53% |

该结果说明 CoT 的影响首先体现在“可观测性”上。Direct answer 往往只有 yes/no、数字或选项，很多推理过程被隐藏；CoT 则会将模型的视觉观察、数量估计、数学公式和中间结论显式写出，从而使 detector 有更多可检查对象。

### 7.2 Exposure effect 与 generation effect

CoT 对幻觉检测结果的影响可以分成两层。

第一是 **exposure effect**。CoT 增加了每条回答中的 claim 数量，即使每个 claim 的错误率不变，response-level 上触发 hallucination 的概率也会增加。换言之，CoT 可能只是把 Direct answer 中隐藏的错误显式暴露出来。

第二是 **generation effect**。CoT 也可能真实引入新的错误。例如，模型为了给出解释，会生成额外视觉描述、语义归因或中间推理。如果这些内容没有被图像、题目或数学规则支持，就会构成新的 hallucination claim。

### 7.3 不同任务中的 CoT 错误机制

在 POPE 中，Direct answer 通常只需要判断图像中是否存在某个对象，因此错误主要集中在 `OBJ`。CoT 会加入更多解释，例如“该物体看起来像 handbag”“图中场景暗示某物存在”等。这些解释可能引入额外的 `IR` 或 `SO`。

在 MathVista 中，CoT 会显式展示视觉读取和数学推理步骤。它能帮助定位错误来源，但也更容易暴露 `ATT` 和 `IR`。例如，模型可能先错误读取图表值或动物数量，再基于错误视觉属性进行形式上合理但实际不可靠的推理。

因此，CoT 的价值是提高可解释性和可诊断性，但代价是增加自动检测难度。报告中不应简单写“CoT 更好”或“CoT 更差”，而应说明 CoT 同时改变了模型生成行为和错误可观测性。

### 9.2 Baseline 与 Zero-shot Judge v2 对齐结果

以下结果以 40 条人工标注样本作为 gold label，并采用 `ann_0020` 复核后的修正版人工标签：该样本从原始 `INC` 幻觉标注修正为 non-hallucination。因此，下表反映的是复核后 Human-as-Judge 口径。

| Detector | TP | FP | TN | FN | Precision | Recall | F1 | Accuracy | Cohen's Kappa | MCC |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| Baseline zero-shot judge | 11 | 11 | 17 | 1 | 0.5000 | 0.9167 | 0.6471 | 0.7000 | 0.4231 | 0.4825 |
| Zero-shot Judge v2 | 9 | 4 | 24 | 3 | 0.6923 | 0.7500 | 0.7200 | 0.8250 | 0.5930 | 0.5940 |

结果显示，v2 相比 baseline 的主要变化是：

- False positives 从 11 降到 4；
- Precision 从 0.5000 提升到 0.6923；
- F1 从 0.6471 提升到 0.7200；
- Accuracy 从 0.7000 提升到 0.8250；
- Cohen's Kappa 从 0.4231 提升到 0.5930，MCC 从 0.4825 提升到 0.5940；
- Recall 从 0.9167 下降到 0.7500，说明 v2 通过更保守的 evidence-boundary 判断减少 false positives，但仍会漏掉部分隐式或细粒度幻觉。

### 9.3 对齐结果解释

Baseline 更像一个“宽松触发”的高召回 detector。它能发现多数人工标注的 hallucination，但也将较多边界样本误判为 hallucination。

Zero-shot Judge v2 则更保守，强调 evidence_source 和 claim-level support status，因此在这 40 条诊断样本上显著减少 false positives。该结果说明 v2 相对更接近当前人工标注集的判断边界，尤其减少了将短答案、合理推理或 reference answer 不可用样本误判为 hallucination 的情况。

不过，v2 的 recall 有所下降，说明它也会漏掉一些隐式或细粒度错误。例如，当模型只输出短答案时，v2 可能因为缺少显式 claim 而漏掉该答案背后的隐含语义断言；当视觉计数或图表读数错误较细微时，judge 也可能过度相信模型自述的 visual evidence。

| MathVista | Qwen | Direct | 5 | 2 | 2 | 0 | 1 | 0.5000 | 0.6667 | 0.5714 | 0.4000 | -0.3636 | -0.4082 |
| MathVista | Qwen | CoT | 5 | 1 | 1 | 3 | 0 | 0.5000 | 1.0000 | 0.6667 | 0.8000 | 0.5455 | 0.6124 |

从这些结果看，POPE Direct 在当前人工样本中没有 false positive 或 false negative，说明对象存在性判断的证据边界相对清晰。POPE CoT 的主要变化是多出 1 个 false positive，说明解释链会引入额外的可疑 claim。MathVista Direct 虽然整体 F1 达到 0.8889，但 Qwen Direct 子组出现 2 个 false positive 和 1 个 false negative，说明复杂视觉数学任务中的判断边界更不稳定。`ann_0020` 修正后，MathVista CoT 的 recall 从原始统计中的 0.5000 提升到 0.6667，MathVista Qwen CoT 子组的 FN 从 1 降为 0；这说明该组原先的低 recall 部分来自人工标注误差，而不是自动 detector 漏检。

因此，Human-as-Judge 结果在当前 40 条诊断样本上支持一个谨慎结论：统一 claim-level judge 在结构上可以覆盖不同任务，但在复杂图文推理任务上的性能边界更明显。

## 10. Disagreement 与 Failure Case 分析

`ann_0020` 修正后，Zero-shot Judge v2 在 40 条 Human-as-Judge 样本中共有 7 条 disagreement，其中包含 3 条 false negative 和 4 条 false positive。为保留复核过程的诊断价值，下表仍列出 `ann_0020`，但将其标记为 human annotation issue，而不是 detector failure。

| Error Pattern | Cases | Dataset | Model | Prompt | Human Label | Judge Label | Direction | 复核后解释 |
|---|---|---|---|---|---|---|---|---|
| 短答案隐含 claim 漏判 | ann_0002 | MathVista | Gemini | Direct | SO | None | FN | Detector 漏掉短答案背后的隐含语义断言 |
| 短答案隐含 claim 漏判 | ann_0014 | MathVista | Qwen | Direct | ATT | None | FN | Detector 未展开 `No` 背后的图表属性比较 claim |
| 视觉属性 / 计数错误漏判 | ann_0010 | MathVista | Gemini | CoT | ATT | None | FN | Detector 过度接受模型自述的 visual evidence |
| 视觉属性 / 计数错误漏判 | ann_0014 | MathVista | Qwen | Direct | ATT | None | FN | 与短答案隐含 claim 漏判重叠，核心是图表属性比较未被验证 |
| 对缺少推理或边界推理过度严格 | ann_0013 | MathVista | Qwen | Direct | None | IR | FP | Detector 将正确短答案误判为 unsupported reasoning |
| 对缺少推理或边界推理过度严格 | ann_0016 | MathVista | Qwen | CoT | None | IR | FP | Detector 对合法几何推理中间步骤要求过严 |
| 模糊估计被过度惩罚 | ann_0015 | MathVista | Qwen | Direct | None | IR | FP | Detector 将可接受近似估计视为无证据断言 |
| 合理视觉推断被过度惩罚 | ann_0037 | POPE | Qwen | CoT | None | IR | FP | Detector 将合理视觉识别依据误判为 unsupported inference |

这些错误模式不是彼此孤立的表面现象，而是来自几个更深层的判断困难：Direct answer 隐藏了答案背后的语义断言，CoT 又把视觉证据、推理步骤和最终答案拆成更长的证据链；同时，judge 既要避免把短答案和合理推理误判为幻觉，又要发现细粒度视觉属性错误。复核 `ann_0020` 后还可以看到，Human-as-Judge 本身也需要校准，不能把人工标签不加检查地视为绝对真值。下表将每类 error pattern 进一步归因到背后的机制。

| Error Pattern | 更深层原因 | 为什么会导致当前方向 | 对 detector / 标注流程的启示 |
|---|---|---|---|
| 短答案隐含 claim 漏判 | Direct answer 只暴露最终答案，没有显式暴露支撑该答案的视觉或语义前提 | v2 为减少 false positive，倾向于不把 answer token 本身当作 hallucination claim，因此可能漏掉短答案背后的不可验证断言 | 需要把短数字、yes/no、选项答案展开为 implicit answer claim，再判断其 evidence_source 是否充分 |
| 视觉属性 / 计数错误漏判 | Judge 可能把模型写出的 visual evidence 当作已验证证据，而不是待核查 claim | 当模型在 CoT 中给出错误计数、颜色、图表值或状态时，后续 reasoning 可能形式上自洽，导致 judge 忽略源头视觉属性错误 | CoT 的 visual evidence 不能直接视为 grounding，应逐条验证其中的数量、属性、OCR 和图表读数 |
| 人工 INC 标注复核误差 | 比较题和填空题需要先解析题干主语、比较方向和答案格式；人工标注也可能过度依赖第一轮直觉 | 原始人工标签将 `ann_0020` 标为 INC，会把 detector 的正确保守判断误计为 false negative；修正后该样本应计为 true negative | Human-as-Judge 应增加复核环节；对方向敏感题型应先做 question-frame parsing，再判定是否内部矛盾 |
| 对缺少推理或边界推理过度严格 | Judge 对“缺少解释”和“证据不足”的边界不稳定，可能把未展开的合理答案当作 unsupported | 这种情况会产生 false positive：模型没有写出完整证据链，但并不等于回答中出现了不被证据支持的事实 claim | 需要区分 missing explanation 与 hallucination；短答案或简洁推理应优先检查是否有明确 contradicted / unverifiable claim |
| 模糊估计被过度惩罚 | 一些视觉任务允许近似判断或粗粒度估计，但 judge 可能按精确读数标准要求模型 | 当题目本身不要求精确数值，或者图像只支持区间判断时，过严格的 evidence boundary 会把可接受估计判为 hallucination | 需要根据任务要求区分 exact-answer claim、approximate claim 和 qualitative claim |
| 合理视觉推断被过度惩罚 | Judge 对“由图像可合理推出”和“超出图像证据的语义外推”边界把握不足 | 合法的场景推断、常识性视觉解释或题干约束推理可能被误判为 SO 或 IR | 需要显式允许由 image、question、choices、math_rule 或 diagram_rule 支持的 legitimate inference，同时继续限制无证据的意图、身份和用途推断 |

因此，这些 disagreement 的根本原因可以概括为四类：第一，**隐含 claim 未展开**，主要影响 Direct answer；第二，**视觉证据未被独立验证**，主要影响 CoT 中的 visual evidence；第三，**证据边界过松或过严**，分别造成 false negative 和 false positive；第四，**人工标注存在边界样本误差**，会影响自动 detector 的对齐统计。后续优化不应只调整标签优先级，而应同时改进 claim extraction、evidence verification、answer-frame consistency 检查和 Human-as-Judge 复核流程。

以下选择三个具有代表性的案例进行深入分析。

### 10.1 Case 1：视觉计数错误导致的 ATT 幻觉（ann_0010）

- Dataset / Model / Prompt：MathVista / Gemini / CoT
- Question：Are most of the animals in the water?
- Model behavior：模型在 visual evidence 中声称陆地上有 16 只动物、水中有 5 只动物，并据此推理大多数动物不在水中。
- Human label：幻觉，`ATT`
- Judge result：非幻觉，false negative

该案例的关键不是模型是否给出 yes/no，而是模型把视觉场景转化为数量证据时出现了属性级错误。CoT 表面上增强了可解释性，因为模型给出了“16 vs 5”的依据；但如果这个计数本身错误，后续推理即使形式上成立，也建立在错误视觉属性上。

Judge 漏判的原因是它可能过度接受了 response 自称的 visual evidence，把“模型说自己看到 16 和 5”当成已经 grounded 的证据，而没有独立验证计数。这说明 MathVista 中的 `ATT` 不只是颜色或图表值，也包括数量、类别和视觉状态。对这类任务，CoT 中的 visual evidence 不能直接视为可靠证据，而应作为待验证 claim。

### 10.2 Case 2：人工 INC 标注复核误差（ann_0020）

- Dataset / Model / Prompt：MathVista / Qwen / CoT
- Question：The derivative of f(x) at x=-2 is ____ that at x=2
- Model behavior：模型围绕 `x=-2` 与 `x=2` 处导数大小进行比较，并给出 final answer。
- Original human label：幻觉，`INC`
- Judge result：非幻觉
- Recheck result：更适合作为人工标注误差，而不是 Zero-shot Judge v2 的 false negative

该案例原本被归为 reasoning-final answer 不一致漏判，但复核后发现，这一判断依赖对题干比较方向和 final answer 语义的严格解释。对于这类填空比较题，必须先明确题干主语、比较对象和答案词所对应的方向；如果没有完成这一步，人工标注者可能把“局部推理表述”和“最终填空答案”之间的关系误读为内部矛盾。

因此，`ann_0020` 更适合说明 Human-as-Judge 的边界：人工标注虽然是校准自动 detector 的重要依据，但并不是绝对真值。尤其在 MathVista 这类方向敏感题型中，人工评审也可能因为题干框架解析不充分而产生错误标签。这个案例提示后续人工标注应加入复核机制：对 greater/less、yes/no、选项映射和填空题，先显式写出题干语义框架，再判断 reasoning 与 final answer 是否真正矛盾。

### 10.3 Case 3：短答案背后的 SO 幻觉（ann_0002）

- Dataset / Model / Prompt：MathVista / Gemini / Direct
- Question：How many people in the image were born after the end of World War II?
- Model behavior：模型只回答 `0`。
- Human label：幻觉，`SO`
- Judge result：非幻觉，false negative

这个案例展示了 Direct answer 的隐蔽性。模型没有显式写出任何推理，因此 Judge 认为没有 checkable claim；但从问题语义看，回答 `0` 实际隐含了一个强断言：图中所有人的出生时间都不晚于二战结束。普通图像通常无法直接提供人物出生年份，除非存在文字、上下文或历史身份信息。因此该回答不是简单的格式化短答，而是基于缺失证据的语义推断。

该案例说明 v2 为减少 false positives 而弱化 final answer 检查后，会产生短答案盲区。改进方向不是回到 baseline 那种简单把 answer mismatch 判为 hallucination，而是为短数字、yes/no 和选择题答案构造 implicit answer claim。例如，`0` 应被展开为“图中没有人出生在二战结束后”，再判断该 claim 是否有足够证据来源。

### 10.4 Failure Case 总结

从以上案例可以看出，最有诊断价值的 failure cases 往往不是简单的“答案错”，而是错误发生在证据链的不同位置：

- 视觉属性读取错误会污染后续推理；
- 短答案会隐藏其背后的不可验证语义断言；
- 方向敏感题型会放大题干框架解析和人工标注复核的重要性；
- 合理数学推理或模糊估计可能被 judge 过度惩罚；
- 弱视觉推断和语义过度推断之间存在边界。

因此，后续优化应同时面向 detector 与标注流程：detector 需要从 response-level 分类进一步走向 claim-level、implicit answer claim、answer-frame consistency 和 evidence-source-level 的联合验证；Human-as-Judge 则需要在 greater/less、yes/no、选项映射和填空题中增加方向敏感复核。

## 11. 跨任务泛化与 XLRS 扩展

### 11.1 统一 Claim-level Judge 的跨任务适用性

本实验中的跨任务泛化不是指“在 POPE 上训练、再迁移到 MathVista”，因为本方法不训练模型。更准确的定义是：同一套 claim extraction、claim type classification、evidence source assignment、support status judgment、taxonomy mapping 和 deterministic aggregation 协议，能否结构化覆盖不同任务中的不同 claim。换言之，这里考察的不是模型参数迁移，而是 **评估协议的结构迁移能力**。

POPE 与 MathVista 的差异如下：

| 分析维度 | POPE | MathVista | 对统一 Judge 的含义 |
|---|---|---|---|
| 任务结构 | Yes/No object presence 判断 | 图文数学推理、图表读取、几何/函数理解 | MathVista 的证据链更长 |
| 主要 claim 类型 | answer_claim、object_claim | reasoning_claim、attribute_claim、answer_claim | 跨任务差异体现在 claim composition |
| 主要 evidence source | image、question | image、math_rule、diagram_rule、question、internal_consistency | MathVista 需要多源证据融合 |
| response 形态 | 短 yes/no 或简短解释 | 数字、选项、填空、公式推理和自然语言解释并存 | MathVista 更依赖 answer-frame binding |
| 主要标签 | `OBJ` | `IR`、`ATT`、`INC` | taxonomy 能覆盖主要错误，但证据边界更复杂 |
| Human alignment 难点 | 合理视觉推断边界 | 短答案隐含 claim、属性读取、方向敏感题型复核 | MathVista CoT 是主要压力测试 |

从 POPE 到 MathVista 的变化，不是简单更换数据集，而是从“对象是否存在”推进到“视觉证据如何支持多步推理”。因此，跨任务泛化需要分成三层理解。

第一层是 **schema-level 泛化**。在 POPE 中，Judge 只需要把回答拆成少量 `answer_claim` 或 `object_claim`，并判断该对象是否被图像和题干支持。例如“图中是否有 handbag”这一类问题，claim、evidence source 和 final label 之间的映射较短：如果 reference/question 指向 handbag，而图像证据不支持或回答方向相反，通常可以直接映射到 `OBJ`。到了 MathVista，同一套 schema 仍然可用，但 claim composition 发生了变化：一条回答可能同时包含图表数值读取、几何关系判断、数学公式使用、单位或选项映射，以及 final answer。此时 claim-level schema 的价值在于它不要求为 MathVista 单独设计一套标签体系，而是把复杂回答拆解为 `attribute_claim`、`reasoning_claim`、`answer_claim` 和 internal-consistency claim，再映射到已有的 `ATT/IR/INC/CI` 等标签。

第二层是 **evidence-source 泛化**。POPE 的证据主要来自 image 与 question，判断边界相对直接；MathVista 的证据来源则是组合式的：图像提供数值、坐标、图形结构或表格内容，题干提供比较方向和答案格式，数学规则决定推理是否合法，diagram rule 决定几何关系能否推出，internal_consistency 决定 visual evidence、reasoning 与 final answer 是否互相支持。也就是说，MathVista 中一个 claim 是否 hallucinated，往往不能只看“图中有没有这个对象”，而要判断该 claim 是否同时满足视觉读取正确、题干绑定正确和数学推理正确。这解释了为什么同一 Judge 在 POPE 上更稳定，而在 MathVista 上更容易出现 disagreement。

第三层是 **aggregation-level 泛化**。Zero-shot Judge v2 的 response-level hallucination 不是由顶层直觉判断决定，而是由 normalized `claim_checks` 聚合得到。这个聚合规则在 POPE 和 MathVista 上是一致的：只要存在关键 unsupported 或 contradicted claim，就可能把整条回答标为 hallucination。但不同任务中“关键 claim”的密度不同。POPE Direct 一条回答通常只有 1 个核心 claim，因此 response-level label 与单个对象判断高度绑定；MathVista CoT 一条回答可能包含多个视觉属性 claim 和多个推理 claim，只要其中一个关键数值读取或推理步骤错误，就会污染后续 reasoning。因此，CoT 在 MathVista 上更容易提高 response-level hallucination rate，并不一定说明模型整体能力下降，而是说明可检查 claim 的数量和证据链长度增加了。

这种跨任务结果说明，Zero-shot Judge v2 具备的是 **结构适用性**，而不是 **性能等价性**。结构适用性意味着同一套 claim-level 协议可以在 POPE 与 MathVista 上运行，输出可比较的 claim 类型、evidence source、support status 和 fine label；性能等价性则要求它在两个任务上都达到相同的一致性和错误边界，这一点当前实验并不支持。Human-as-Judge 结果也体现了这一点：POPE Direct 的证据边界清晰，人工和自动 Judge 更容易一致；MathVista 中 Qwen Direct、MathVista CoT 等子组更容易出现 false positive、false negative 或人工标注复核问题。

更深层地看，跨任务泛化的困难来自 **证据边界从离散对象判断变成连续证据链判断**。POPE 的核心问题是对象是否存在，错误通常集中为 `OBJ`；MathVista 的核心问题是模型如何把图像中的视觉属性转化为数学或语言答案，错误可能先表现为 `ATT`，再传播为 `IR`，最后在 final answer 中表现为错误答案或潜在 `INC`。这意味着 taxonomy 虽然可以覆盖两类任务，但标签解释必须结合任务机制：同样是 hallucination，POPE 中更多是视觉 grounding 失败，MathVista 中更多是视觉读取、规则使用和答案框架绑定的联合失败。

因此，本实验对跨任务泛化的结论应保持分层表述：

1. **可以泛化的是评估框架。**  
   claim extraction、evidence-source assignment、support-status 判断和 deterministic aggregation 可以从 POPE 迁移到 MathVista，不需要为每个任务重新定义幻觉检测流程。

2. **需要适配的是证据解释。**  
   MathVista 需要更强的图表值读取、几何规则、数学规则和 answer-frame consistency 检查；POPE 则更需要区分合理视觉推断和对象存在性幻觉。

3. **不能直接泛化的是指标表现。**  
   POPE 上的高一致性不能推出 MathVista 上也会高一致；MathVista 中的高 hallucination rate 也不能简单和 POPE 数字横向比较，因为两者的 claim 数量、证据链长度和可观测错误密度不同。

4. **Human-as-Judge 仍是跨任务校准的必要环节。**  
   自动 Judge 可以提供统一输出格式，但不同任务的人类判断边界不同。尤其在 MathVista 中，短答案隐含 claim、方向敏感填空题和图表属性读取都需要人工复核来校准自动 detector 的误差类型。

综上，Zero-shot Judge v2 的跨任务价值不在于“在所有数据集上得到同样高的指标”，而在于它提供了一种统一诊断语言：可以把 POPE 的对象存在性错误、MathVista 的视觉数学推理错误，以及后文 XLRS 的遥感地物错误，都放到同一个 claim-level evidence checking 框架下比较。但报告也必须承认，随着任务从 object grounding 走向 multi-source reasoning，Judge 的视觉复核能力、数学规则理解能力和答案框架绑定能力会成为主要瓶颈。

### 11.2 XLRS-bench 遥感扩展

第二个加分项选择 remote sensing / XLRS-bench，但本节关注的重点不是单纯“换一个遥感数据集”，而是遥感超分辨率图像如何适配当前多模态大模型输入这一问题。遥感图像通常具有大幅面、高分辨率、小目标密集和空间关系复杂等特点；而多模态大模型的图像输入存在尺寸、token budget 和视觉编码分辨率限制。若直接缩放整幅遥感图像，小目标、道路边界、船只、油罐等局部细节容易被压缩掉；若直接输入超分辨率图，又可能超出模型输入预算，或让模型把插值增强产生的边缘、纹理和伪影误认为真实地物。因此，XLRS-bench 在本实验中的作用，是用遥感任务说明“高分辨率 / 超分辨率视觉证据进入 MLLM 前必须经过受控预处理”，并观察这种预处理会如何影响幻觉检测结果。

当前 XLRS pilot 完成 4 个实验组，每组 100 条样本，并使用 Gemini-judge v2 进行自动评测。需要强调的是，XLRS 结果是 automated diagnostic extension，不等同于人工标注真值。下表中的 hallucination rate 表示“100 条回答中被自动 judge 判为包含至少一个 unsupported 或 contradicted claim 的比例”，不是人工验证后的真实幻觉率。该 pilot 结果用于说明遥感场景本身会显著放大 `ATT`、`SPA` 和 `IR` 类错误；后文 XLRS-SR original/sr/paired 对照则进一步分析如何通过配合 SR 的图像预处理适配 MLLM 输入。

| Run | n | Hallucination | Rate | Answer correct | Correct rate | 主要 fine labels |
|---|---:|---:|---:|---:|---:|---|
| `gemini_direct` | 100 | 95 | 0.95 | 5 | 0.05 | `ATT=50`, `SPA=49`, `IR=38`, `OBJ=13`, `INC=3`, `CI=1` |
| `gemini_cot` | 100 | 89 | 0.89 | 14 | 0.14 | `IR=54`, `ATT=48`, `SPA=26`, `OBJ=16`, `CI=8`, `INC=5` |
| `qwen_direct` | 100 | 98 | 0.98 | 2 | 0.02 | `SPA=45`, `IR=35`, `ATT=31`, `OBJ=12`, `CI=6`, `INC=4` |
| `qwen_cot` | 100 | 96 | 0.96 | 5 | 0.05 | `IR=68`, `ATT=34`, `OBJ=21`, `SPA=21`, `CI=13`, `INC=1` |

从 pilot 结果看，XLRS 场景中的错误主要集中在 `ATT`、`SPA` 和 `IR`。模型容易误读遥感图像中的数量、地物属性和空间关系，也容易从有限视觉证据跳到不充分支持的场景结论。

### 11.3 XLRS-SR：original / sr / paired 对照

除基础 pilot 外，实验还构造了 XLRS-SR 扩展，用于比较原图、超分图和 global-local paired 输入下的遥感回答与 hallucination 诊断。

该实验包含：

```text
3 variants: original, sr, paired
2 models: gemini, qwen
2 prompts: direct, cot
```

XLRS-SR 的设计目标不是证明“超分一定提升模型性能”，而是构造一个可控的遥感证据干预实验：在题目、参考答案、模型、prompt 和 judge 协议不变的前提下，只改变图像证据呈现方式，观察 hallucination 类型是否随视觉输入变化而变化。

更具体地说，本实验中的 SR 方法设计不是为了恢复真实高分辨率遥感图，也不是为了追求视觉观感上的清晰度，而是为了解决“遥感超分辨率图如何进入当前 MLLM 输入接口”这一问题。设计时需要同时满足五个约束：第一，不能让派生图像超出模型的图像尺寸和 token budget；第二，不能因为整体缩放而丢失机场、船只、道路、油罐等小目标细节；第三，不能只放大局部而丢掉遥感图像的全局空间布局；第四，必须把 SR 插值产生的边缘增强、纹理增强和潜在伪影显式纳入 prompt 约束；第五，所有派生输入都要保留可追溯 metadata，使后续 hallucination 诊断能够知道模型看到的是原图、SR 图还是 global-local paired 图。

因此，XLRS-SR 被设计成一个 input-budget-aware global-local SR pipeline。它先用低风险、可复现的 Lanczos 2× 插值构造 SR 证据，再用最大边长限制把 SR 图控制在模型可接受的输入预算内；随后从原始大幅面遥感图中选择高细节 tile，把全局原图、全局 SR 图和局部 SR tile 组织为同一张 contact sheet。这样做的核心目的，是在不改变题目和模型的情况下，把“全局场景理解”和“小目标局部证据”同时暴露给 MLLM，并让 judge 能观察不同证据呈现方式下的 hallucination profile。

#### 11.3.1 图像前处理与三种输入构造

XLRS-SR 的图像前处理由 `src/datasets/build_xlrs_global_local_sr.py` 和 `src/datasets/prepare_xlrs_sr.py` 完成。它不是重新训练或调用深度学习超分模型，而是在统一输入预算下构造三种可对照的图像证据形式：`original`、`sr` 和 `paired`。

1. **original 输入：保留全局遥感图像。**  
   `original` variant 使用 XLRS 评测样本中的全局图像路径作为 `image_path`。在已有处理数据中，样本 metadata 保留 `original_image_path`、`resized_for_inference=true` 和 `max_side=2048` 等字段，用于说明该图像已经按推理输入预算进行尺寸约束。该输入提供完整场景布局，适合观察模型在原始全局证据下的小目标计数、空间关系和地物属性判断。

2. **sr 输入：Lanczos 2× 上采样并限制最大边长。**  
   `sr` variant 读取原始遥感图像，将其转换为 RGB 后使用 `Image.Resampling.LANCZOS` 做 2× 插值上采样，即 `sr_scale=2`。随后用 `max_side=2048` 限制派生图像最长边，避免超出多模态模型输入预算，并以 JPEG quality=90 保存到 `data/raw/xlrs_bench/images_sr_lanczos_2x_capped/`。因此，本实验中的 SR 更准确地说是 **Lanczos interpolation-based super-resolution / sharpening-style evidence intervention**，不是 Real-ESRGAN、SwinIR 等学习式超分模型。报告中不能把它表述为“恢复真实高分辨率细节”，只能说它增强了边缘和纹理可见性，同时也可能引入插值伪影。

   在本实验中选择 Lanczos 作为图像证据增强方法，主要有四点优势。第一，它是确定性的传统插值算法，同一输入和参数会得到稳定输出，便于复现实验和控制变量。第二，相比最近邻或双线性插值，Lanczos 对边缘、细线结构和局部纹理的保留能力更强，更适合遥感图像中道路边界、船只轮廓、建筑物边缘和密集小目标区域的可见性增强。第三，它不会像学习式超分模型那样引入额外训练数据先验，因此较少产生“模型生成出来”的语义细节，更适合用于 hallucination 评估中的证据边界控制。第四，它计算成本低、实现简单，可以批量处理 XLRS 样本，并与 `max_side=2048` 的输入预算约束结合，形成可复现、可追溯的 SR-aware 输入构造流程。

3. **paired 输入：global-local contact sheet。**  
   `paired` variant 不是简单把原图和 SR 图分别输入模型，而是生成一张 global-local 组合图。脚本先把原始图像划分为 `4 × 4` 网格，对每个 tile 计算 detail score：灰度边缘响应均值加灰度标准差，即 `edge_mean + gray_stddev`。然后选择 detail score 最高的 `top_k_tiles=4` 个局部区域，对这些 crop 做 2× Lanczos 上采样，并压缩到 `512 × 512` tile panel。最终 contact sheet 的上半部分左侧放 `GLOBAL ORIGINAL`，右侧放 `GLOBAL SR-LANCZOS`，下半部分放被选中的高细节局部 tile，例如 `TILE_R3C3`、`TILE_R0C2`。完整 paired 图保存到 `data/raw/xlrs_bench/images_paired_global_local_lanczos_2x/`，对应 tile 的 bbox、detail score 和来源写入 `tile_manifests_lanczos_2x/*.json`。

4. **metadata 与证据协议。**  
   三种 variant 都保留相同的问题、选项和参考答案，只改变 `image_path` 与 metadata 中的图像证据字段。metadata 会记录 `xlrs_sr_variant`、`evidence_protocol=input_budget_aware_global_local_sr`、`sr_method=lanczos`、`sr_scale=2`、`input_budget_capped=true`、`derived_image_max_side=2048`、`global_panel_size=1024`、`tile_panel_size=512`、`tile_grid_size=4`、`top_k_tiles=4`，以及 `sr_image_path`、`paired_image_path` 和 `tile_manifest_path`。这保证了后续模型回答和 judge 结果可以追溯到具体图像前处理方式。

5. **Prompt 约束。**  
   XLRS-SR 的 prompt 明确要求模型只依据可见遥感证据回答；如果小目标、细纹理、计数或边界证据不足，应回答 `uncertain`。CoT prompt 还要求模型区分 `[original full image]`、`[SR full image]` 和 `[SR tile r2c3]` 等证据来源，并进行 SR consistency check：说明哪些结论由原图确认，哪些只在 SR 图或局部 tile 中可见，以及哪些细节可能是超分伪影。prompt 同时明确禁止把 SR sharpening、checkerboard pattern、artificial texture 或 enhanced edges 当作真实道路、建筑、车辆、植被或边界。

基于上述设计，`original` 用于保留全局上下文，`sr` 用于测试插值增强后的细节可见性，`paired` 用于同时提供全局布局和高细节局部证据。三者的差异是输入证据呈现方式，而不是题目、参考答案、模型或 judge 协议的差异。

| Variant | Model | Prompt | n | Hallucination rate | Correct rate | 主要 fine labels |
|---|---|---|---:|---:|---:|---|
| original | gemini | direct | 100 | 0.52 | 0.03 | `ATT=30`, `IR=8`, `SPA=7`, `CI=7` |
| original | gemini | cot | 100 | 0.87 | 0.15 | `ATT=45`, `IR=45`, `OBJ=19`, `SPA=18`, `CI=12` |
| original | qwen | direct | 100 | 0.59 | 0.00 | `ATT=28`, `IR=15`, `CI=12`, `SPA=8` |
| original | qwen | cot | 100 | 0.97 | 0.06 | `IR=60`, `ATT=42`, `SPA=16`, `OBJ=14`, `CI=13` |
| sr | gemini | direct | 100 | 0.52 | 0.03 | `ATT=29`, `IR=11`, `SPA=8`, `CI=5` |
| sr | gemini | cot | 100 | 0.88 | 0.12 | `IR=45`, `ATT=43`, `SPA=21`, `OBJ=20`, `CI=14` |
| sr | qwen | direct | 100 | 0.53 | 0.01 | `ATT=29`, `CI=9`, `IR=8`, `SPA=7` |
| sr | qwen | cot | 100 | 0.93 | 0.08 | `IR=57`, `ATT=38`, `SPA=21`, `OBJ=11`, `CI=9`, `SO=2` |
| paired | gemini | direct | 100 | 0.41 | 0.03 | `ATT=33`, `IR=5`, `SPA=3`, `CI=2`, `INC=1` |
| paired | gemini | cot | 100 | 0.88 | 0.14 | `IR=49`, `ATT=38`, `SPA=18`, `OBJ=15`, `CI=13` |
| paired | qwen | direct | 100 | 0.48 | 0.00 | `ATT=34`, `CI=7`, `IR=5`, `SPA=3`, `INC=1` |
| paired | qwen | cot | 100 | 0.96 | 0.04 | `IR=57`, `ATT=41`, `CI=18`, `SPA=12`, `OBJ=9`, `SO=1` |

在 Gemini-judge v2 的自动诊断结果中，可以观察到以下趋势：

1. **SR alone 的收益有限。**  
   仅做 2x Lanczos 超分没有稳定降低 hallucination rate。Gemini direct 从 original 到 sr 保持 0.52；Qwen direct 从 0.59 降到 0.53，但改善有限。

2. **paired 输入在 direct prompt 下可能更有帮助。**  
   Gemini direct 从 original 0.52 降到 paired 0.41；Qwen direct 从 original 0.59 降到 paired 0.48。相比只提供原图或 SR 图，把全局图、SR 图和局部高细节 tile 组织到同一输入中，可能更有利于模型同时利用全局布局和局部细节。

3. **CoT 下 SR/paired 没有稳定降低 hallucination rate。**  
   CoT 仍然会生成更多中间 claim，因此即使视觉输入更丰富，`IR`、`ATT` 和 `OBJ` 仍然高频出现。

因此，XLRS-SR 的谨慎结论是：SR alone 在当前自动诊断设置下没有稳定降低 hallucination；global-local paired 输入在 direct prompt 下可能减少部分视觉证据错误；CoT 虽可能提升回答完整性，但也会扩大 hallucination 暴露面。

## 12. 方法局限性

### 12.1 Human-as-Judge 样本量有限

当前人工标注集包含 40 条样本，满足考核中 20–50 条小规模人工验证集的要求，也足以进行 disagreement pattern 分析。但从统计意义上看，40 条样本不足以支持强显著性结论。尤其在按 dataset、model、prompt 和 fine label 拆分后，每个子组样本数更小。

因此，Human-as-Judge 结果应作为 detector 设计的校准证据，而不是最终 benchmark 排名。

### 12.2 MLLM-as-a-Judge 本身仍可能有偏差

Zero-shot Judge v2 虽然引入了 claim-level schema 和 deterministic aggregation，但底层判断仍依赖多模态 LLM。Judge 可能出现 modality neglect，即过度依赖文本回答的流畅性或自洽性，而没有充分复核图像证据。它也可能存在 verbosity / CoT bias：长 CoT 回答提供更多 claim，既可能暴露错误，也可能因为结构完整而影响 judge 判断。

### 12.3 细粒度视觉属性验证仍然困难

MathVista 中的图表值、数量、颜色、角度、斜率和 OCR-like evidence，以及 XLRS 中的小目标、空间布局和地物属性，对自动 judge 都是挑战。当前 v2 能让判断过程更透明，但不能保证底层视觉证据一定正确。

### 12.4 CoT 同时改变模型行为和错误可观测性

本实验中 CoT 的 hallucination rate 全面高于 Direct，但该结果不能直接解释为“CoT 使模型真实更容易幻觉”。CoT 同时改变了生成行为和错误可观测性。它可能生成新的错误，也可能只是把 Direct answer 中隐藏的中间证据和推理显式暴露出来。

### 12.5 Taxonomy 标签存在边界重叠

`OBJ/ATT/SPA/IR/CI/INC/SO` 能覆盖当前实验中的主要错误，但实际案例中标签边界可能重叠。例如图表数值读取错误可以标为 `ATT`，但如果错误数值导致后续推理错误，也可能表现为 `IR`。短答案语义外推可以标为 `SO`，但如果它源于视觉属性误读，也可能与 `ATT` 相关。

因此，fine label 应被理解为主要错误来源，而不是互斥的唯一解释。

### 12.6 XLRS 扩展缺少人工验证

XLRS 与 XLRS-SR 结果目前定位为 automated diagnostic extension，没有对应的 Human-as-Judge gold label。因此，报告不能把 XLRS 的 hallucination rate 表述为人工验证后的真实幻觉率。它更适合用于展示方法在遥感领域中的潜在适用性、错误标签覆盖能力和后续人工标注优先级。

## 13. 未来工作

后续可以从以下方向继续改进：

1. **扩大人工标注规模。**  
   将 Human-as-Judge 样本从 40 条扩展到更大规模，并覆盖更多 dataset、model 和 prompt 组合。

2. **引入多名人工标注者。**  
   计算 inter-annotator agreement，进一步区分 detector 错误与人工标注边界不一致。

3. **增强 implicit answer claim reconstruction。**  
   对短数字、yes/no、选择题答案自动展开隐含 claim，减少 Direct answer 下的漏判。

4. **加入 reasoning-to-answer entailment check。**  
   针对 greater/less than、填空、选择题等方向敏感任务，显式检查 reasoning 是否真的支持 final answer。

5. **引入外部视觉或结构化工具。**  
   对图表值、OCR、数量、角度、空间关系等细粒度 evidence，可结合 OCR、图表解析、目标检测或几何分析工具辅助判断。

6. **扩展更多任务。**  
   后续可加入 ChartQA、OCRBench、LogiCOCR、完整 XLRS-bench 或医学图像任务，检验统一 claim-level judge 在更多证据边界下的稳定性。

7. **比较多个 Judge 模型。**  
   使用多 judge ensemble 或 self-consistency 方法，分析不同 judge 模型之间的一致性和偏差。

## 14. 结论

本报告围绕多模态大模型幻觉检测与评估，完成了从幻觉定义、自动检测方法、主实验评估到 Human-as-Judge 对齐和跨任务扩展的完整分析。

第一，本实验构建了面向多模态回答的 claim-level 幻觉定义，并使用 `OBJ/ATT/SPA/IR/CI/INC/SO` 七类细粒度标签覆盖对象、属性、空间、推理、上下文、内部一致性和语义过度推断等主要错误类型。

第二，实验实现了 baseline zero-shot judge 与 Zero-shot Judge v2。v2 通过 evidence_source、support_status、non_claim / not_applicable 和 deterministic aggregation，将检测过程从 response-level summary 推进到 taxonomy-constrained claim-level evidence checking。

第三，在 POPE 与 MathVista 主实验中，POPE Direct 主要表现为 `OBJ` 对象幻觉，而 MathVista 更容易出现 `IR` 和 `ATT`。这说明不同任务的 hallucination profile 与其证据结构密切相关：对象存在性任务证据边界较短，图文数学推理任务则需要多源证据融合。

第四，CoT 在所有任务和模型组合中提高了 response-level hallucination rate，但这一现象应结合 claim exposure 理解。CoT 增加了每条回答中可检查 claim 的数量，使错误更容易被定位，同时也可能生成新的 unsupported reasoning 或 semantic overreach。

第五，在 40 条 Human-as-Judge 诊断样本上，Zero-shot Judge v2 相比 baseline 明显减少 false positives，并在 Accuracy、Precision、F1、Cohen's Kappa 和 MCC 上取得更好的对齐结果。不过，v2 仍存在短答案隐含 claim 漏判、视觉属性细节漏判等问题；同时，`ann_0020` 的复核说明方向敏感题型还需要更严格的 Human-as-Judge 标注校准。

第六，跨任务与 XLRS 扩展表明，统一 claim-level judge 在结构上具有迁移潜力，但当证据边界从常规 VQA 扩展到图文数学推理和遥感图像理解时，自动 judge 的视觉复核能力、多源证据融合能力和语义边界判断仍需要进一步增强。

综上，Zero-shot Judge v2 更适合作为一种 diagnostic hallucination detector：它可以批量发现多模态大模型的潜在幻觉模式，支持模型、任务和 prompt 之间的对比，并辅助人工分析；但若用于正式高风险评测，还需要更大规模人工标注、多 judge 校准和外部视觉工具增强。
