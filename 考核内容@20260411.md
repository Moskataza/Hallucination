# 题目：多模态大模型的幻觉检测与评估

---

## 一、研究内容

针对多模态大模型（MLLM），研究其在视觉问答（VQA）或图文推理任务中的幻觉现象，重点关注：

- 幻觉的定义与分类  
- 幻觉的自动检测方法  
- 幻觉检测结果与人类判断的一致性（human-as-judge）

## 十、当前实验进展更新：Zero-shot Judge v2 与 Human-as-Judge 对齐

### -1. 实验设计总览

本实验围绕“多模态大模型在 VQA 与图文数学推理任务中的幻觉如何被自动检测，以及自动检测结果是否与人类判断一致”展开。整体设计采用 **2 个任务 × 2 个模型 × 2 种回答方式 × 2 个 Judge 版本** 的对照结构，并额外加入 Human-as-Judge 小规模人工验证集，用于评估自动 Judge 的可信度。

### -0. 幻觉类型定义与示例

本实验将多模态大模型幻觉定义为：模型回答中出现了与图像、题干、选项、参考答案、数学/图形规则或回答内部逻辑不一致，或者缺乏足够证据支持的可检查断言。为便于自动检测与人工标注对齐，本文采用四类粗粒度幻觉，并进一步映射到 `OBJ/ATT/SPA/IR/CI/INC/SO` 七个细粒度标签。

| 粗粒度类型 | 对应细标签 | 核心判断依据 |
|---|---|---|
| 视觉事实幻觉 | `OBJ`、`ATT`、`SPA` | 对象、属性、数量、空间关系是否被图像支持 |
| 推理幻觉 | `IR`、`CI` | 推理步骤、数学规则、因果关系是否成立 |
| 内部一致性幻觉 | `INC` | reasoning、visual evidence 和 final answer 是否前后一致 |
| 语义过度推断幻觉 | `SO` | 是否从图像过度外推身份、意图、年代、用途或场景语义 |

#### -0.1 视觉事实幻觉（Factual / Visual Hallucination）

视觉事实幻觉指模型对图像中可直接验证的对象、属性、数量或空间关系作出错误或无依据的陈述。它主要覆盖 `OBJ`、`ATT` 和 `SPA`。

- **`OBJ` 对象幻觉示例 1：** POPE 问题询问图中是否有 handbag，如果图中没有 handbag，但模型回答 yes，则属于 object presence hallucination。
- **`OBJ` 对象幻觉示例 2：** 图像中只有狗和人，模型却声称图中有 cat，属于不存在对象的虚构。
- **`ATT` 属性幻觉示例 1：** MathVista 中模型声称陆地上有 16 只动物、水中有 5 只动物，但人工检查认为数量读取错误，属于数量/属性误读。
- **`ATT` 属性幻觉示例 2：** 图表中 Firebrick 与 Midnight Blue 数值相等或接近，但模型断言其中一个明显更大，属于图表属性或数值读取错误。
- **`SPA` 空间幻觉示例 1：** 模型声称物体 A 在物体 B 左侧，但图像中 A 实际位于 B 右侧。
- **`SPA` 空间幻觉示例 2：** 模型声称车辆在道路中央，但图像显示车辆停在道路边缘或停车区，属于空间位置判断错误。

#### -0.2 推理幻觉（Reasoning Hallucination）

推理幻觉指模型从图像、题干或选项到答案的中间推理不成立，或者使用了不被题目条件、数学规则、图形规则支持的因果或逻辑关系。它主要覆盖 `IR` 和 `CI`。

- **`IR` 无效推理示例 1：** 几何题中模型在没有足够对应边/角条件的情况下，直接推出两条线段相等，并据此计算答案。
- **`IR` 无效推理示例 2：** 模型在 MathVista 中给出面积、斜率或比例结论，但中间公式与图中数值不匹配，或缺少必要计算步骤。
- **`CI` 因果推理示例 1：** 模型看到图中有人拿着手机，就断言“这个人正在给朋友打电话”，但图像只能支持“拿着手机”，不能支持具体行为目的。
- **`CI` 因果推理示例 2：** 遥感图像中模型看到大面积建筑，就断言该区域一定是工业污染源，但图像证据不足以支持这种因果结论。

#### -0.3 内部一致性幻觉（Internal Consistency Hallucination）

内部一致性幻觉指模型回答内部前后矛盾，或 reasoning 与 final answer 的语义方向不一致。它对应 `INC`。

- **`INC` 示例 1：** `ann_0020` 中，模型推理认为 `f'(2)` 大于 `f'(-2)`，但最终答案 `greater` 在题干语义中表示 `f'(-2) > f'(2)`，reasoning 与 final answer 方向相反。
- **`INC` 示例 2：** 模型前文说图中没有目标对象，后文却用该对象作为证据回答 yes，属于同一回答内部自相矛盾。
- **`INC` 示例 3：** 模型 visual evidence 中声称图表最高柱为蓝色，但 reasoning 中又把红色柱作为最高值进行计算，导致证据链内部不一致。

#### -0.4 语义过度推断幻觉（Semantic Overreach）

语义过度推断幻觉指模型从图像可见信息过度外推到身份、意图、年代、用途、社会语义或不可见背景知识，且缺少足够证据支持。它对应 `SO`。

- **`SO` 示例 1：** `ann_0002` 中，问题问图中有多少人出生在二战结束之后，模型回答 `0`，但图像本身通常无法直接证明人物出生年代。
- **`SO` 示例 2：** 模型看到一个人穿白色衣服，就断言其职业是医生；除非图像或题干提供明确证据，否则这是身份语义过度推断。
- **`SO` 示例 3：** 遥感图像中模型仅凭建筑外观断言该区域是军事基地或工业园区，而没有足够地物、文字或上下文证据支持。

这种定义方式既满足考核中至少两类幻觉类型与示例的要求，也与后续自动 Judge 的 claim-level taxonomy 保持一致：`OBJ/ATT/SPA` 主要用于视觉事实错误，`IR/CI` 用于推理错误，`INC` 用于回答内部矛盾，`SO` 用于语义外推。

#### -1.1 研究问题

实验主要回答四个问题：

1. **RQ1：不同任务中的幻觉类型是否不同？**  POPE 主要考察 object presence / absence，因此预期以 `OBJ` 为主；MathVista 需要图表读取、视觉属性判断和数学推理，因此预期 `ATT` 与 `IR` 更突出。
2. **RQ2：CoT 是否会改变幻觉率和幻觉类型分布？**  对比 direct answer 与 evidence-grounded CoT，观察 CoT 是降低幻觉、暴露更多中间错误，还是引入额外 reasoning / semantic hallucination。
3. **RQ3：不同模型的 hallucination profile 是否不同？**  比较闭源模型 Gemini 与开源/开放接口模型 Qwen-VL 在相同任务和 prompt 设置下的 hallucination rate 与标签分布。
4. **RQ4：自动 Judge 与人类判断是否一致？**  使用 40 条人工标注样本作为 gold label，评估 zero-shot judge v1 / v2 与 human-as-judge 的一致性，并分析 disagreement case。

#### -1.2 任务、模型与实验变量

| 维度 | 设计 |
|---|---|
| 主任务 1 | POPE，通用 VQA / object presence yes-no 判断 |
| 主任务 2 | MathVista，视觉数学推理与图文理解 |
| 加分扩展 | XLRS-bench pilot-ready 接入，用于遥感领域扩展，但当前不伪造定量结果 |
| 模型 | Gemini 作为闭源模型代表；Qwen-VL 作为开源/开放接口模型代表 |
| Prompt 设置 | Direct answer；Evidence-grounded CoT |
| Detector | Baseline zero-shot judge；Zero-shot Judge v2 |
| Human validation | 40 条固定人工标注样本，覆盖 dataset、model、prompt_type |

其中自变量包括 dataset、model、prompt_type 和 detector version；因变量包括 response-level hallucination label、fine-grained hallucination taxonomy、hallucination rate、accuracy / precision / recall / F1、Cohen's Kappa、MCC 以及 claim-level exposure 指标。

#### -1.3 实验流程

整体流程分为六步：

```text
统一数据格式
→ 生成模型回答（Gemini / Qwen × Direct / CoT）
→ 解析回答中的 final answer、visual evidence、reasoning
→ 使用 zero-shot judge v1 / v2 进行自动幻觉检测
→ 汇总 response-level 与 claim-level 指标
→ 使用人工标注集验证自动 Judge 与 human-as-judge 的一致性
```

具体而言，POPE 与 MathVista 样本先被统一为 EvalSample schema，保证问题、选项、reference answer、图像路径和 metadata 的字段一致。随后每个样本分别使用 direct prompt 和 evidence-grounded CoT prompt 生成模型回答。CoT prompt 要求模型显式写出 visual evidence、reasoning 和 final answer，以便后续 claim-level judge 能检查中间证据链。

自动检测阶段先运行 baseline zero-shot judge，再运行 Zero-shot Judge v2。baseline 作为高召回、低精度的初始方法；v2 则引入 evidence_source、support_status、not_applicable / non_claim 和 deterministic aggregation，用于减少误报并提升与人类判断的一致性。

#### -1.4 对照组设计

实验包含三类核心对照：

1. **Prompt 对照：Direct vs CoT。**  同一 dataset 和 model 下比较 direct answer 与 evidence-grounded CoT，观察 CoT 对 hallucination rate、claim 数量和幻觉类型分布的影响。
2. **模型对照：Gemini vs Qwen。**  同一 dataset 和 prompt 下比较两个模型，分析闭源模型与开源/开放接口模型在视觉 grounding 和图文推理上的差异。
3. **Detector 对照：Baseline vs v2。**  在相同 human annotation gold labels 上比较两个 Judge 版本，判断 v2 是否减少 false positives、提高 F1 / Accuracy / Kappa / MCC。

这种设计的优点是每个结论都有明确参照：CoT 影响不是跨模型比较得出，而是在同一模型内比较；模型差异不是跨 prompt 比较得出，而是在同一 prompt 下比较；v2 改进不是只看全量 hallucination rate，而是通过 human-as-judge gold labels 验证。

#### -1.5 评估指标设计

实验采用 response-level、label-level、claim-level 和 human-alignment 四组指标：

| 指标类型 | 指标 | 用途 |
|---|---|---|
| Response-level | Hallucination Rate | 衡量某一实验组合中被判为幻觉的样本比例 |
| Binary alignment | Accuracy, Precision, Recall, F1 | 衡量自动 Judge 与人工 binary label 的一致性 |
| Agreement metrics | Cohen's Kappa, MCC | 在类别不平衡情况下补充衡量一致性和相关性 |
| Label-level | Fine-label distribution, per-label alignment | 分析 `OBJ/ATT/SPA/IR/CI/INC/SO` 的分布和标签级错误 |
| Claim-level | Claims / Resp., Unsupported Claim Ratio, Label-triggering Unsupported Ratio | 区分 CoT 导致的真实错误增加与 claim exposure 增加 |
| Error analysis | FP / FN disagreement cases | 解释 Judge 与人类判断不一致的具体原因 |

其中 Accuracy 与 F1 满足考核中“至少 2 种评估指标”的基本要求；Hallucination Rate 用于模型/任务/prompt 对比；Cohen's Kappa 与 MCC 用于增强 human-as-judge 一致性分析；claim-level 指标用于解释 CoT 幻觉率上升的机制。

#### -1.6 人工标注与一致性验证设计

Human-as-Judge 部分使用 40 条固定验证样本，覆盖 POPE、MathVista、Gemini、Qwen、Direct 和 CoT。人工标注内容包括：

- binary hallucination label；
- coarse taxonomy（Factual / Logical / None）；
- fine taxonomy（`OBJ`、`ATT`、`SPA`、`IR`、`CI`、`INC`、`SO`）；
- human rationale；
- answer correctness。

自动 Judge 的输出通过 versioned key 与人工标注样本对齐，避免 baseline 与 v2 结果互相覆盖。对齐后计算 TP / FP / TN / FN、Accuracy、Precision、Recall、F1、Cohen's Kappa 和 MCC，并导出 disagreement cases。所有 disagreement 都进行逐例分析，用于判断错误来自视觉误读、推理边界、短答案隐含 claim、还是 Judge 与人工的 hallucination 定义边界不同。

#### -1.7 设计合理性与局限

该实验设计的优势在于结构清晰、变量可控，并且符合“不训练模型，只允许 inference、prompt engineering 和简单工具调用”的约束。通过 dataset、model、prompt 和 detector 四个维度的交叉对照，可以同时回答模型行为差异、CoT 影响和 Judge 可靠性问题。

局限性也需要明确：第一，Human-as-Judge 样本量为 40 条，适合作为诊断性验证，但不足以支持强统计显著性结论；第二，自动 Judge 仍依赖多模态 LLM 自身的视觉理解能力，因此可能把模型 response 中错误的 visual evidence 当成真实证据；第三，MathVista 中部分答案 reference answer 为 `UNAVAILABLE`，使 final answer correctness 与 hallucination 判断之间存在边界争议。因此，报告中的结论应强调趋势、机制解释和 error pattern，而不是过度声称绝对性能。

### 0. Baseline Zero-shot Judge 的制作方式

此前 baseline detector 是一个 prompt-based GPT judge 方法，不训练额外模型，而是直接调用多模态 LLM 作为 hallucination judge。每条 model response 会和对应的图像、问题、选项、reference answer、解析出的 visual evidence、reasoning、final answer 一起输入 judge prompt。

baseline judge 的输入结构包括：

```text
Dataset / Task Type / Question / Choices / Reference Answer
Metadata / Model Response
Parsed Visual Evidence / Parsed Reasoning / Parsed Final Answer
Taxonomy Definition
```

baseline 使用固定 taxonomy：

- Factual Hallucination：`OBJ`、`ATT`、`SPA`；
- Logical Hallucination：`IR`、`CI`、`INC`、`SO`。

其核心流程是：

```text
理解问题要求
→ 从 model response 中抽取 explicit verifiable claims
→ 将 claim 分类为 object / attribute / spatial / reasoning / causal / inconsistency / semantic / answer claim
→ 判断每个 claim 是 supported / contradicted / unverifiable
→ 将有问题的 claim 映射到 OBJ / ATT / SPA / IR / CI / INC / SO
→ 程序从 claim_checks 推导 response-level is_hallucination、hallucination_labels、primary_label 等字段
```

baseline 输出为 strict JSON，模型主要返回字段包括：

```text
answer_correct
is_hallucination
hallucination_vector
primary_label
coarse_labels
unsupported_visual_claim
confidence
claim_checks
explanation
```

其中 `hallucination_labels` 不要求模型直接输出，而由程序从 `claim_checks` 中的 `support_status` 与 `fine_label` 确定性推导。

随后程序会把 judge 返回的 JSON 解析为 detector result，并写入每个实验组合对应的 detector 文件，例如：

```text
outputs/detector_results/one_tenth_pope_gemini_direct_zero_shot.jsonl
outputs/detector_results/one_tenth_pope_gemini_cot_zero_shot.jsonl
outputs/detector_results/one_tenth_mathvista_qwen_cot_zero_shot.jsonl
```

baseline 的优点是实现简单、无需训练、可解释性较强，并且已经能输出 claim-level 判断和细粒度 taxonomy。但它也存在明显问题：

- 缺少显式 `evidence_source` 字段，judge 不一定清楚每个 claim 应该依据 image、question、reference answer、math rule 还是 internal consistency；
- 不支持 `non_claim` 与 `not_applicable`，容易把格式文本、简短回答或不可检查内容误当成可疑 claim；
- 对 `unverifiable` 的处理偏严格，尤其在 reference answer 为 `UNAVAILABLE` 时，容易把 answer-only difference 判成 hallucination；
- 顶层 `is_hallucination` 与 claim-level checks 的一致性约束不够强，可能出现 raw judge summary 与 normalized detector 不一致；
- 对 POPE 简短 yes/no 和 MathVista 合理图形/数学推理的非幻觉边界不够清晰。

因此 baseline 在当前 human-as-judge 标注集上的表现呈现“高召回、低精度”的特点：

| Detector | TP | FP | TN | FN | Precision | Recall | F1 | Accuracy |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| Baseline zero-shot judge | 11 | 11 | 16 | 2 | 0.5000 | 0.8462 | 0.6286 | 0.6750 |

这说明 baseline 能发现大部分 human-labeled hallucination，但会产生较多 false positives。Zero-shot Judge v2 的设计正是针对这些问题进行改进。

### 0.1 Baseline Judge 与 Zero-shot Judge v2 的差别

需要注意的是，baseline judge 本质上也是 zero-shot judge，可以理解为 `zero-shot judge v1`；当前新版方法是 `zero-shot judge v2`。两者都不训练模型，均属于 prompt-based GPT judge 方法，但 v2 在 claim-level 约束、证据来源标注和程序化归一化方面更严格。

| 维度 | Baseline Judge / Zero-shot Judge v1 | Zero-shot Judge v2 |
|---|---|---|
| 方法性质 | Prompt-based zero-shot GPT judge | Prompt-based zero-shot GPT judge |
| 是否训练模型 | 否 | 否 |
| 判断粒度 | Response-level 判断为主，同时输出 claim checks | Taxonomy-constrained claim-level evidence checking |
| Claim 抽取 | 抽取 explicit verifiable claims | 抽取 atomic checkable claims，并区分 factual / reasoning / semantic / answer / non-claim |
| Evidence 约束 | 主要依赖 image evidence、question semantics 和 reference context，证据来源不显式 | 每个 claim 必须标注 `evidence_source`，如 image、question、choices、reference_answer、math_rule、diagram_rule、internal_consistency |
| Claim 状态 | supported / contradicted / unverifiable | supported / contradicted / unverifiable / not_applicable |
| 非事实内容处理 | 没有显式 `non_claim` 类型 | 增加 `non_claim`，格式文本、空泛表达、不可检查内容不触发 hallucination |
| 不适用内容处理 | 没有显式 `not_applicable` | 增加 `not_applicable`，answer token、格式文本等可被排除出 hallucination 判断 |
| Final answer 处理 | final answer 会进入 claim checks，错误答案容易影响 hallucination 判断 | final_answer correctness 单独判断；incorrect final answer alone 不自动等于 hallucination |
| Reference answer 不可用 | 虽有规则提醒，但仍可能把不确定性或 answer-only mismatch 判为 hallucination | 明确 `reference_answer=UNAVAILABLE` 不能作为 contradiction evidence |
| 合理推理处理 | 对 MathVista 中未直接写在图中的推理较容易判为 unsupported / IR | 允许基于 math_rule、diagram_rule、question constraints 的 legitimate inference |
| 聚合方式 | 较依赖 judge 返回的 top-level `is_hallucination` 和 label summary | 不直接信任 top-level 输出，只基于 normalized `claim_checks` 聚合 response-level 结果 |
| 一致性审计 | 缺少显式 raw-vs-normalized mismatch 记录 | 增加 `summary_consistent_with_claims` 和 `raw_normalized_mismatch` |
| 主要错误倾向 | 高 recall、低 precision，false positives 较多 | false positives 明显减少，precision / F1 / accuracy 更高，但 recall 有所下降 |

从实验结果看，baseline 更像一个“宽松触发”的高召回 detector，而 v2 更像一个“证据约束更强”的平衡 detector。v2 的核心改进不是简单换 prompt，而是把 judge 输出限制为可归一化的 claim-level evidence-checking 结构，并由程序进行 deterministic aggregation。

### 1. Zero-shot Judge v2 方法更新

当前已完成第二版 zero-shot judge detector。v2 不再把 LLM judge 的顶层 `is_hallucination` 作为唯一可信输出，而是采用 taxonomy-constrained claim-level evidence-checking 流程：

```text
固定幻觉定义与标签体系
→ 从完整 model response / CoT 中抽取 atomic checkable claims
→ 为每个 claim 标注 evidence_source
→ 判断 supported / contradicted / unverifiable / not_applicable
→ 将问题 claim 映射到 OBJ / ATT / SPA / IR / CI / INC / SO
→ 单独判断 final_answer correctness
→ 只基于 normalized claim_checks 聚合 response-level detector 结果
```

v2 支持的主要字段包括：

- `claim_checks`：claim-level 判断来源；
- `evidence_source`：`image`、`question`、`choices`、`reference_answer`、`math_rule`、`diagram_rule`、`internal_consistency`、`none`；
- `support_status`：`supported`、`contradicted`、`unverifiable`、`not_applicable`；
- `summary_consistent_with_claims`：检查 judge 顶层摘要是否与 claim-level 结果一致；
- `raw_normalized_mismatch`：记录 raw judge 输出与 normalized detector 输出是否不一致。

该设计重点解决 v1 的主要问题：judge 过于严格，容易把简短回答、合理图形/数学推理、或 reference answer 不可用导致的不确定性误判为 hallucination。

#### 6.4 原因讨论与改进方向

1. **v2 的保守聚合降低了误报，但带来了短答案盲区。** v2 不再简单把 final answer mismatch 当成 hallucination，这使 false positives 明显减少；但当模型只输出 `0`、`No` 或一个选项时，答案本身仍隐含了可验证的视觉、数学或语义判断。若 Judge 只寻找显式 claim，就会漏掉这类 implicit answer claim。

2. **MathVista 的证据边界比 POPE 更复杂。** POPE 主要检查 object presence，而 MathVista 同时涉及图表值读取、数量统计、几何对应关系、函数图像斜率和比较题语义解析。`ATT` 漏判说明 Judge 可能接受 response 自称的 visual evidence，却没有真正验证图表值、数量或视觉属性是否正确。

3. **Judge 与人工对 `unverifiable` 和 `hallucination` 的边界不同。** 在 `ann_0013` 和 `ann_0016` 中，人工更重视答案或推理是否正确、是否可由题目条件推出；Judge 则因为 direct answer 缺少推理或中间推导没有完全展开而判为 IR。在 `ann_0015` 中，人工认为年龄差估计本身存在模糊性，Judge 则把具体数值视为无证据断言。这说明 `unverifiable` 不应在所有场景下自动等价于 hallucination。

4. **CoT 提高了可解释性，也提高了判断难度。** CoT 暴露更多中间 claim，使 reasoning hallucination 更容易被发现；但 CoT 也会引入弱推断、模糊视觉描述和中间推理链。若 Judge 过度惩罚弱推断，会产生 false positive；若缺少 question-frame-aware 的 final answer 检查，又会漏掉 `ann_0020` 这类 reasoning 与最终答案方向相反的 INC。

后续改进可以集中在五个方面：第一，对短数字、yes/no 和选择题答案增加 `implicit_answer_claim` 检查；第二，对比较题和填空题增加 question-frame-aware final answer consistency 检查；第三，对 MathVista 中的 `ATT` claim，尤其是数量、图表值、颜色/类别和视觉属性，增加更严格的视觉验证；第四，把“缺少推理导致不可验证”和“明确幻觉推理”区分开，避免正确 direct answer 被误判为 IR；第五，在 POPE 中优先检查 object presence 与 reference answer 的一致性，对 `appears to be`、`resembles`、`could be` 等谨慎措辞不应直接判为 unsupported hallucination。

### 7. 当前结论与后续重点

当前 v2 zero-shot judge 已满足本考核中“自动化幻觉检测方法”和“Human-as-Judge 一致性验证”的核心要求：

- 幻觉 taxonomy 已明确覆盖 factual / visual / reasoning / consistency 等类型；
- 自动检测方法具备可解释的 claim-level evidence-checking 流程；
- 已在 POPE 与 MathVista 上完成 Gemini 与 Qwen 的 detector 输出；
- 已使用 40 条人工标注样本完成 human-as-judge alignment；
- 已比较 Accuracy、Precision、Recall、F1、Cohen's Kappa、MCC、Hallucination Rate 等指标；
- 已识别 v2 相比 baseline 的主要优势和剩余 failure modes。

后续报告和 PPT 中应重点呈现：

1. v2 detector 的核心方法不是简单 prompt 打分，而是 taxonomy-constrained claim-level judge；
2. v2 显著降低 false positives，更符合 human-as-judge 的判断边界；
3. CoT 会提高可检查 claim 数量，因此更容易暴露 reasoning hallucination，但也会提高 detector 判断难度；
4. MathVista 是后续优化重点，尤其是 `ATT`、`INC` 和 borderline `IR`；
5. 当前方法不训练模型，仅依赖 prompt engineering 和 deterministic normalization，符合实验约束。

- MathVista Direct 中 `reasoning_claim` 与 `attribute_claim` 已经占较大比例。Gemini Direct 中 `reasoning_claim` 占 38.35%，`attribute_claim` 占 13.64%；Qwen Direct 中 `reasoning_claim` 占 25.50%，`attribute_claim` 占 13.85%。
- MathVista CoT 进一步暴露大量中间视觉、数值与推理 claim。Gemini CoT 中 `reasoning_claim` 占 39.26%，`attribute_claim` 占 29.64%；Qwen CoT 中 `reasoning_claim` 占 40.33%，`attribute_claim` 占 28.51%。

Evidence source 分布也说明两个任务的证据边界不同：

- POPE 更集中于 `image` evidence。Gemini CoT 中 `image` 占 77.61%，Qwen CoT 中 `image` 占 82.69%。
- MathVista 同时依赖 `image`、`math_rule`、`diagram_rule`、`question` 和 `internal_consistency`。例如 Gemini Direct 中 `image` 占 27.44%，`math_rule` 占 24.32%，`diagram_rule` 占 17.77%；Gemini CoT 中 `image` 升至 51.00%，`math_rule` 为 22.86%，`diagram_rule` 为 12.77%。

输出格式方面，所有 v2 detector 文件均未出现 `claim_checks` 缺失、非 list、非法 `claim_type`、非法 `support_status`、非法 `evidence_source` 或非法 `fine_label`。这说明统一 JSON schema 在两个任务上结构稳定。不过，`raw_normalized_mismatch` 与 `summary_consistent_with_claims=false` 仍然存在，尤其在 CoT 设置下更多，说明 deterministic normalization 是必要的。

#### RQ3：CoT 幻觉率上升究竟来自什么？

RQ3 不只比较 response-level hallucination rate，而是同时报告 claim exposure：Claims per Response、Unsupported Claims per Response、Unsupported Claim Ratio，以及更贴近 detector 聚合逻辑的 Label-triggering Unsupported Claim Ratio。后者只统计 `support_status` 为 `contradicted` / `unverifiable` 且 `fine_label` 属于 `OBJ/ATT/SPA/IR/CI/INC/SO` 的 claim，排除不会触发 hallucination label 的 answer-only 或 non-claim 情况。

关键观察是：CoT 显著增加了每条回答中可检查 claim 的数量。

| Dataset | Model | Direct Claims / Resp. | CoT Claims / Resp. | Δ Claims / Resp. | Direct Label-triggering Unsupported Ratio | CoT Label-triggering Unsupported Ratio | Δ Response Hallucination Rate |
|---|---|---:|---:|---:|---:|---:|---:|
| POPE | Gemini | 1.12 | 5.57 | +4.45 | 6.06% | 6.70% | +14.22% |
| POPE | Qwen | 1.13 | 6.60 | +5.47 | 6.93% | 5.94% | +12.33% |
| MathVista | Gemini | 2.50 | 8.15 | +5.66 | 11.96% | 7.89% | +16.15% |
| MathVista | Qwen | 1.85 | 6.86 | +5.01 | 14.89% | 13.30% | +22.37% |

如果使用所有 unsupported claim 作为分母，CoT 下 unsupported claim ratio 也并不总是上升：POPE Gemini 从 6.36% 到 6.97%，POPE Qwen 从 7.43% 降到 6.27%，MathVista Gemini 从 15.28% 降到 8.49%，MathVista Qwen 从 21.34% 降到 14.15%。

因此，更严谨的结论是：

> CoT 显式暴露了更多中间视觉和推理 claim，从而增加了 Judge 可检查的潜在幻觉点。现有 response-level 结果尚不能完全区分“模型真实变得更容易幻觉”和“claim 暴露数量增加”这两种因素。从当前 claim-level 指标看，CoT 的 response-level hallucination rate 上升，很大程度上与 claims per response 显著增加有关；每个 claim 的 unsupported ratio 并不总是上升，甚至在多个设置下降。

这使 CoT 分析从简单的“CoT 幻觉率更高”推进到更有解释力的 claim exposure analysis。

#### RQ4：跨任务结果说明了什么？

前面的 RQ1–RQ3 说明统一 claim-level 协议在结构上可以覆盖 POPE 与 MathVista，但加分项的重点不只是“方法能跨任务运行”，更重要的是比较跨任务结果揭示了哪些任务差异、错误机制和泛化边界。

| 分析维度 | POPE 结果特征 | MathVista 结果特征 | 对统一 Judge 的含义 |
|---|---|---|---|
| 任务结构 | Yes/No object presence 判断为主，Direct 下每条回答约 1.12–1.13 个 claim | 图文数学推理、图表读取和几何/函数理解混合，Direct 下每条回答约 1.85–2.50 个 claim | 同一协议可覆盖两类任务，但 MathVista 的证据链更长 |
| 主要 claim 类型 | Direct 主要是 `answer_claim` 与 `object_claim`；CoT 后增加 `reasoning_claim` 和 `semantic_claim` | Direct 已包含较多 `reasoning_claim` 与 `attribute_claim`；CoT 后进一步暴露大量数值、属性和推理 claim | 跨任务差异主要体现在 claim composition，而不只是 hallucination rate |
| 主要证据来源 | 以 `image` evidence 为核心，判断目标对象是否存在 | 同时依赖 `image`、`math_rule`、`diagram_rule`、`question` 和 `internal_consistency` | MathVista 需要多源证据融合，Judge 更容易在 evidence_source 分配上出错 |
| Response-level 幻觉率 | Direct 较低（6.78%–7.67%），CoT 升至约 20%–21% | Direct 已较高（18.87%–23.15%），CoT 升至 35.02%–45.53% | 复杂图文推理任务会显著放大幻觉暴露与 Judge 判断难度 |
| Human-as-Judge 对齐 | POPE Direct 在 40 条人工样本中达到 1.0000 Accuracy；POPE CoT 为 0.9000 | MathVista Direct 为 0.9000，MathVista CoT 降至 0.7000 | 跨任务泛化不是均匀的，MathVista CoT 是当前统一 Judge 的主要压力测试 |
| 主要失败模式 | POPE 中主要风险是合理视觉推断被过度惩罚，如 handbag case | MathVista 中集中出现短答案隐含 claim、属性/计数漏判、合法推理被误判和 answer-frame inconsistency | 统一 taxonomy 足够覆盖错误类型，但不同任务需要不同的证据校准策略 |

除了并列比较 POPE 与 MathVista，还可以把跨任务分析理解为一个从 **POPE → MathVista** 的能力压力递进。POPE 主要检验最基础的视觉 grounding：图像中是否存在某个对象，模型是否把不存在的对象说成存在，或把存在的对象漏掉。这个任务对应的是“对象层”的幻觉检测，证据链相对短，通常可以被简化为 image evidence 与 yes/no answer 的一致性检查。

MathVista 则是在 POPE 的对象 grounding 之上继续增加三层难度。第一，它要求模型读取视觉属性和数值关系，例如数量、长度、颜色、图表高度、斜率或几何标注；第二，它要求模型把这些视觉证据接入数学或图形规则，例如面积公式、全等关系、函数导数和比例计算；第三，它要求模型把推理结果正确映射回自然语言题干和 final answer，例如 greater/less than、yes/no 或填空方向。因此，POPE → MathVista 的变化不是简单换数据集，而是从“对象是否存在”推进到“视觉证据如何支持多步推理”。

这个递进逻辑解释了为什么统一 Judge 在 POPE 上表现更稳定，而在 MathVista 上暴露更多边界问题。如果一个 detector 在 POPE 上有效，只能说明它较好处理了 object-level grounding；它迁移到 MathVista 后，还必须处理 attribute grounding、math/diagram rule grounding、internal consistency 和 answer-frame binding。当前结果正体现了这一点：POPE Direct 的 hallucination rate 只有 6.78%–7.67%，Human-as-Judge 对齐达到 1.0000 Accuracy；而 MathVista Direct 已升至 18.87%–23.15%，MathVista CoT 进一步升至 35.02%–45.53%，且人工对齐中 MathVista CoT Accuracy 降至 0.7000。这说明跨任务泛化的真正难点不是 schema 是否能运行，而是 Judge 能否随着任务从 object grounding 进入 visual reasoning 后继续保持证据判断稳定。

从方法设计看，Zero-shot Judge v2 与跨任务迁移之间的关系可以理解为“固定判断框架 + 可变证据边界”。固定部分包括 atomic claim 抽取、`OBJ/ATT/SPA/IR/CI/INC/SO` taxonomy、`support_status` 判断和 deterministic aggregation；这些设计保证了同一个 detector 不需要为 POPE 和 MathVista 分别重写评估逻辑。可变部分则是每个 claim 应该依赖什么证据：POPE 中多数 claim 只需要 `image` 与 yes/no answer 的一致性，而 MathVista 中 claim 可能需要同时依赖 `image`、`question`、`math_rule`、`diagram_rule` 和 `internal_consistency`。

因此，当前方法的跨任务迁移不是把 POPE 上的 object-hallucination 规则直接套到 MathVista，而是把更抽象的“claim → evidence source → support status → taxonomy label”判断流程迁移过去。这个设计的优点是任务无关性较强：只要新任务中的回答可以被拆成可检查 claim，就能进入同一套 Judge 协议。它也解释了为什么 v2 在两个任务上都没有出现 schema-level failure，说明方法框架本身具有结构迁移能力。

但结果也表明，结构迁移不等于性能迁移。POPE 到 MathVista 的递进暴露了 v2 的深层边界：当证据从单一 image grounding 扩展到多源推理证据时，Judge 必须判断哪些视觉属性可信、哪些数学规则适用、reasoning 是否真的 entail final answer，以及 short answer 是否隐含不可见语义 claim。换言之，v2 的方法设计解决了“如何用同一格式表达不同任务中的幻觉”，但尚未完全解决“如何在不同任务的证据边界下同样可靠地验证这些 claim”。这也是跨任务分析最重要的发现：迁移失败的位置本身反过来揭示了方法需要增强的模块，例如 implicit answer claim reconstruction、attribute verification 和 reasoning-to-answer entailment check。

从结果看，POPE 与 MathVista 的差异不是简单的“一个容易、一个困难”。更准确地说，POPE 的幻觉检测主要依赖视觉对象 grounding：Judge 需要判断目标对象是否存在，以及模型是否虚构了对象。只要图像证据和 reference answer 较清晰，`OBJ` 类型就较容易与人工判断对齐。因此 POPE Direct 的 human-alignment 表现最好，也符合其 claim 结构简单、证据来源集中的特点。

MathVista 则表现出更复杂的跨任务挑战。它的 hallucination rate 更高，尤其 Qwen CoT 达到 45.53%，并不只是因为模型“更差”，而是因为 MathVista 同时要求模型完成视觉属性读取、图表/几何结构理解、数学规则调用、自然语言题干绑定和最终答案映射。任何一个环节出错，都可能表现为 `ATT`、`IR`、`INC` 或 `SO`。因此，MathVista 对 unified claim-level judge 的压力主要来自 evidence boundary 的复杂性，而不是 taxonomy 本身无法覆盖。

跨任务结果还说明，CoT 的影响具有任务依赖性。在 POPE 中，CoT 主要把原本简短的 yes/no 判断扩展为对象描述、场景解释和弱语义推断，因此新增的风险集中在 `IR` 和 `SO`。在 MathVista 中，CoT 则把隐藏的视觉数值读取和数学推理过程显式化，因此更容易暴露 `ATT` 和 `IR`。这意味着 CoT 的“幻觉率上升”在两个任务中机制不同：POPE 更偏向语义扩展和解释过度，MathVista 更偏向中间证据链变长后暴露视觉属性和推理错误。

因此，本加分项的结论是：Zero-shot Judge v2 具备跨任务结构泛化能力，因为同一套 claim-level schema 可以稳定运行于 POPE 和 MathVista，且输出格式没有出现 schema-level failure；但它的性能泛化并不均匀。POPE 上的成功主要说明该协议适合 object-grounding 类任务，MathVista 上的错误则揭示出统一 Judge 在多源证据融合、隐含答案 claim 重构和 reasoning-to-answer entailment 上仍有明显边界。换言之，跨任务分析的价值不只是证明“一个 Judge 可以用于两个数据集”，而是通过两个任务的结果差异定位了下一步最需要优化的证据类型和失败模式。

### 9. 重点分析：CoT 影响、模型间差异与 Failure Cases

本节对应考核中的重点分析要求，目标不是只报告 hallucination rate，而是解释：CoT 为什么会改变幻觉暴露方式，不同模型的错误结构有什么差异，以及典型 failure case 中错误是如何产生的。

#### 9.1 CoT 对幻觉的影响

从 response-level detector 结果看，CoT 在所有 dataset × model 设置下都会提高被判定的 hallucination rate：

| Dataset | Model | Direct Hallucination Rate | CoT Hallucination Rate | Δ CoT - Direct | Direct Main Labels | CoT Main Labels |
|---|---|---:|---:|---:|---|---|
| POPE | Gemini | 6.78% | 21.00% | +14.22% | OBJ=61 | OBJ=95, IR=63, SO=17 |
| POPE | Qwen | 7.67% | 20.00% | +12.33% | OBJ=69 | OBJ=95, SO=42, IR=35 |
| MathVista | Gemini | 18.87% | 35.02% | +16.15% | IR=81, ATT=10 | IR=103, ATT=46, OBJ=13 |
| MathVista | Qwen | 23.15% | 45.53% | +22.37% | IR=95, ATT=12 | IR=144, ATT=55, OBJ=20 |

注：Δ 使用未四舍五入的原始 hallucination rate 计算；若直接用表中两位小数相减，MathVista Qwen 会显示为 +22.38%。

但这一现象不能简单解释为“CoT 一定让模型更容易幻觉”。结合 claim exposure 指标可以看到，CoT 首先显著增加了每条回答中可检查 claim 的数量：POPE Gemini 从 1.12 增至 5.57，POPE Qwen 从 1.13 增至 6.60；MathVista Gemini 从 2.50 增至 8.15，MathVista Qwen 从 1.85 增至 6.86。也就是说，CoT 把原本隐藏在 final answer 后面的视觉观察、数值读取和推理步骤显式写了出来，使 Judge 有更多可检查对象。

因此，CoT 对幻觉检测的影响可以分成两层：

1. **暴露效应（exposure effect）**：CoT 增加 claims per response，即使每个 claim 的错误比例不变，response-level 上只要任一关键 claim 被判为 unsupported / contradicted，就更容易触发 hallucination label。这解释了为什么 CoT 的 response hallucination rate 全面高于 direct。
2. **生成效应（generation effect）**：CoT 也可能真实引入新的错误。POPE CoT 中新增大量 `IR` 和 `SO`，说明模型在解释 object presence 时会生成额外语义推断；MathVista CoT 中 `ATT` 从 Gemini Direct 的 10 增至 46、Qwen Direct 的 12 增至 55，说明中间推理会迫使模型显式读取图表值、数量和视觉属性，从而暴露更多 attribute-level 错误。

按幻觉类型看，CoT 的影响不是均匀的：

- **OBJ**：在 POPE 中，direct 主要就是 object presence 判断，因此 `OBJ` 是主导错误；CoT 下 `OBJ` 仍然存在，但不再是唯一问题，因为解释链引入了更多 reasoning 和 semantic claim。
- **ATT**：在 MathVista 中，CoT 明显提高 `ATT` 暴露量，尤其是图表值、数量、颜色、长度、角度、斜率等视觉属性。CoT 不一定制造这些错误，但会把模型的视觉读取过程写出来，使错误更可见。
- **IR**：MathVista 的 `IR` 在 direct 和 CoT 中都很高，说明数学图文推理本身是主要风险源。CoT 下 `IR` 进一步上升，表明中间推导步骤越多，出现无效推理、跳步推理或证据不足推理的机会越多。
- **SO / INC**：POPE CoT 中 `SO` 增加，说明模型会从“是否存在某物”扩展到用途、身份、语义场景等更高层解释；MathVista CoT 中 `INC` 虽然总体数量不大，但典型性很强，因为它反映的是 reasoning 与 final answer 的语义方向不一致。

结论上，CoT 的价值是提高可解释性和可诊断性，但代价是提高检测难度。Direct answer 的错误常被压缩成一个短答案，难以定位错误来源；CoT 能暴露错误来自视觉读取、数学推理还是语义解释，但也会产生更多边界型 claim，使 Judge 更容易出现 false positive 或 false negative。

#### 9.2 模型间对比分析

模型间差异在 POPE 上较小，在 MathVista 上更明显。

| Dataset | Prompt | Gemini Hallucination Rate | Qwen Hallucination Rate | Difference |
|---|---|---:|---:|---:|
| POPE | Direct | 6.78% | 7.67% | Qwen +0.89% |
| POPE | CoT | 21.00% | 20.00% | Gemini +1.00% |
| MathVista | Direct | 18.87% | 23.15% | Qwen +4.28% |
| MathVista | CoT | 35.02% | 45.53% | Qwen +10.51% |

在 **POPE Direct** 中，Gemini 和 Qwen 的幻觉率接近，且主要标签都集中在 `OBJ`。这说明对于 yes/no object presence 任务，两者的主要错误来源相似：是否正确识别图中目标对象。POPE CoT 下，两者 hallucination rate 仍接近，但标签结构开始分化：Gemini CoT 的 `IR` 更多（63 vs 35），Qwen CoT 的 `SO` 更多（42 vs 17）。这表明 Gemini 的 CoT 更容易在解释链中产生可疑推理步骤，而 Qwen 的 CoT 更容易加入语义性、场景性或对象用途相关的额外断言。

在 **MathVista** 中，模型差异明显扩大。Qwen 在 direct 和 CoT 下的 hallucination rate 都高于 Gemini，尤其 Qwen CoT 达到 45.53%，比 Gemini CoT 高 10.51 个百分点。MathVista 需要同时处理视觉读取和数学推导，因此模型差异不只来自最终答案正确率，还来自中间证据组织能力：

- Gemini 在 MathVista CoT 中也有大量 `IR` 和 `ATT`，说明闭源模型并不能避免图文数学推理中的幻觉；
- Qwen 的 MathVista CoT 中 `IR=144`、`ATT=55`，说明它在展开推理链时更容易暴露无效推理和视觉属性误读；
- Qwen Direct 的幻觉率已经高于 Gemini Direct，说明即使不展开 CoT，其 final answer 背后的视觉-数学推理也更容易出现错误或被 Judge 判为 unsupported。

从任务角度看，POPE 更像是视觉 grounding 能力测试，模型差异主要体现在 object presence；MathVista 则是视觉 grounding、数值读取、数学规则和语言语义的组合测试，因此更容易放大模型间差异。换言之，模型在 POPE 上表现接近，并不意味着它们在复杂图文推理上的 hallucination profile 也接近。

#### 9.3 Failure Case 深入分析

以下 failure cases 选择了三类具有代表性的错误：视觉属性读取错误、reasoning-final answer 不一致、以及短答案隐含语义幻觉。它们分别对应 MathVista 中最关键的 `ATT`、`INC` 和 `SO` 风险边界。

##### Case 1：视觉计数错误导致的 ATT 幻觉（ann_0010）

- Dataset / Model / Prompt：MathVista / Gemini / CoT
- Question：Are most of the animals in the water?
- Model behavior：模型在 visual evidence 中给出具体计数，声称陆地上有 16 只动物、水中有 5 只动物，并据此推理大多数动物不在水中。
- Human label：幻觉，`ATT`
- Judge result：非幻觉，false negative

这个案例的关键不是模型最终是否选择了 yes/no，而是模型把视觉场景转化为数量证据时发生了属性级错误。CoT 看似增强了可解释性，因为模型给出了“16 vs 5”的依据；但如果这个计数本身错误，后续推理即使形式上正确，也建立在错误视觉属性上。

错误来源可以拆成三步：首先，模型需要从图中分割动物个体；其次，需要判断每个动物位于水中还是岸上；最后，需要比较两类数量。任一步视觉 grounding 出错，都会导致最终判断不可靠。Judge 漏判的原因是它过度相信 response 自称的 visual evidence，把“模型说自己看到 16 和 5”当成已经 grounded 的证据，而没有独立验证计数。

这说明 MathVista / Chart-like 任务中的 `ATT` 不只是颜色或数值读取，也包括数量、位置状态和类别属性。对这类任务，CoT 中的 visual evidence 不能直接视为可靠证据；Judge 必须把它当作待验证 claim。

##### Case 2：推理与最终答案方向相反的 INC 幻觉（ann_0020）

- Dataset / Model / Prompt：MathVista / Qwen / CoT
- Question：The derivative of f(x) at x=-2 is ____ that at x=2
- Model behavior：模型推理中说 x=2 处曲线更陡，因此 x=2 处导数大于 x=-2 处导数；但 final answer 填写 `greater`。
- Human label：幻觉，`INC`
- Judge result：非幻觉，false negative

这个案例体现了 CoT 的另一类风险：局部推理可以看起来合理，但最终答案与题干语义框架不一致。题干要求比较的是 `f'(-2)` 相对于 `f'(2)`，如果推理认为 `f'(2)` 更大，那么 final answer 应该表达 `f'(-2)` smaller than `f'(2)`。模型输出 `greater`，等价于 `f'(-2) > f'(2)`，与前面的推理方向相反。

错误来源不是单纯的视觉误读，而是 language frame binding 失败：模型没有把推理结论正确映射回填空句子的主语和比较方向。这类错误在数学填空、比较题、选择题中很常见，因为模型需要同时维护视觉判断、数学关系和自然语言句式。

Judge 漏判说明当前 v2 对 internal consistency 的检查还不够 question-frame-aware。它检查了“导数代表斜率”“x=2 更陡”这些局部 claim，却没有执行最后一步：验证 final answer 是否被前面的 reasoning entail。后续 detector 应显式加入 reasoning-to-answer entailment check，尤其针对 greater than / less than / yes-no / fill-in-blank 这类方向敏感题型。

##### Case 3：短答案背后的 SO 幻觉（ann_0002）

- Dataset / Model / Prompt：MathVista / Gemini / Direct
- Question：How many people in the image were born after the end of World War II?
- Model behavior：模型只回答 `0`。
- Human label：幻觉，`SO`
- Judge result：非幻觉，false negative

这个案例展示了 direct answer 的隐蔽性。模型没有显式写出任何推理，因此 Judge 认为没有 checkable claim；但从问题语义看，回答 `0` 实际隐含了一个强断言：图中所有人的出生时间都不晚于二战结束。普通图像通常无法直接提供人物出生年份，除非存在文字、上下文或历史身份信息。因此该回答不是简单的格式化短答，而是基于缺失证据的语义推断。

错误来源是 semantic overreach：模型把可见年龄、外貌或场景线索过度外推为出生年代判断。人工标为 `SO` 是合理的，因为问题要求的证据超出图像直接可见内容。

这个案例说明 v2 为减少 false positives 而弱化 final answer 检查后，会产生短答案盲区。改进方向不是回到 baseline 那种简单把 answer mismatch 判为 hallucination，而是为短数字、yes/no 和选择题答案构造 implicit answer claim。例如，`0` 应被展开为“图中没有人出生在二战结束后”，再判断该 claim 是否有足够证据来源。

#### 9.4 综合结论

综合 CoT、模型对比和 failure cases，可以得到三个结论。

第一，CoT 并不是单纯“更好”或“更差”的 prompt 策略。它显著提高 claim exposure，使幻觉更容易被定位到 object、attribute、reasoning 或 semantic 层面；但它也增加了 response-level 被判 hallucination 的概率，并引入更多边界型推理和弱视觉推断。

第二，模型间差异具有任务依赖性。POPE 上 Gemini 与 Qwen 的 overall hallucination rate 接近，但 MathVista 上 Qwen 明显更高，尤其在 CoT 设置下。这说明复杂图文数学推理比普通 object-presence VQA 更能区分模型的视觉-语言-推理综合能力。

第三，最有诊断价值的 failure cases 往往不是简单的“答案错”，而是错误发生在证据链的不同位置：视觉属性读取错误会污染后续推理；question-frame 绑定失败会导致 reasoning 与 final answer 相反；短答案会隐藏其背后的不可验证语义断言。因此，后续 detector 优化应从 response-level 分类进一步走向 claim-level、answer-frame-level 和 evidence-source-level 的联合验证。

#### 9.5 方法局限性与适用边界

虽然 Zero-shot Judge v2 相比 baseline 明显提升了 precision、F1、accuracy、Kappa 和 MCC，但它仍然不是一个“绝对可靠”的幻觉判定器。它更适合作为一种可解释的自动分析工具，用于批量发现潜在 hallucination pattern，并辅助 human-as-judge 进行诊断，而不应被理解为完全替代人工标注的最终裁判。

##### 9.5.1 Judge 本身仍依赖多模态 LLM 的视觉理解能力

v2 的核心是让 Judge 对 model response 中的 atomic claims 进行 evidence checking。但 Judge 仍然是一个多模态 LLM，它对图像、图表、几何图形和场景细节的理解也可能出错。因此，v2 的 claim-level 结构能让判断过程更透明，却不能保证底层视觉证据一定正确。

`ann_0010` 就体现了这一限制：模型在 CoT 中给出动物数量，Judge 接受了该 visual evidence，但人工认为计数错误。这说明当 response 自称“我看到了 X 个对象”时，Judge 可能没有真正完成独立视觉复核，而是受到 response wording 的影响。对于 MathVista 中的图表值、数量、斜率、角度、颜色和空间位置，这种风险尤其明显。

因此，v2 更擅长检测语言层面和结构层面的 unsupported / contradicted claim；对需要高精度视觉感知的 `ATT`、`SPA` 或 OCR-like claim，它仍可能漏判。

##### 9.5.2 `unverifiable` 与 `hallucination` 的边界存在主观性

幻觉并不总是等同于“缺少完整证据”。有些回答虽然没有展开推理，但结果正确；有些估计虽然无法精确验证，但在人类判断中是可接受的近似。v2 为了提高可解释性，会把 claim 标为 supported、contradicted、unverifiable 或 not_applicable，但 `unverifiable` 是否应触发 hallucination label，本身依赖任务语境。

例如 `ann_0013` 中，Qwen 直接给出正确面积 `285 square inches`，Judge 因缺少推理而误判为 `IR`；`ann_0015` 中，年龄差估计约 20 岁，人工认为图像本身难以精确判断，因此不应严格视为幻觉。这说明 v2 对“没有证据证明”和“有证据反驳”之间的区分仍不够细。

后续应将 unsupported 分成更细的层级，例如：

- **contradicted claim**：与图像、题干或 reference 明确冲突，应强触发 hallucination；
- **unverifiable but risky claim**：缺少证据且语气确定，可触发 hallucination；
- **unverifiable but acceptable estimate**：任务本身允许估计或模糊判断，不应自动判为 hallucination；
- **correct direct answer without reasoning**：缺少解释不等于推理幻觉。

##### 9.5.3 短答案和选择题答案会隐藏隐含 claim

v2 为减少 baseline 的 false positives，避免简单将 final answer mismatch 判为 hallucination。这一设计提升了 precision，但也带来短答案盲区。Direct prompt 下，模型可能只输出 `Yes`、`No`、`0` 或一个选项；这些回答表面上没有显式 reasoning，但实际上隐含了完整的视觉或语义判断。

`ann_0002` 中的 `0` 隐含“图中没有人出生在二战之后”；`ann_0014` 中的 `No` 隐含“Firebrick 不大于 Midnight Blue”。如果 Judge 只检查显式 claim，就会认为没有可检测对象，从而漏掉人类可以识别的幻觉。

因此，v2 的一个重要局限是：它对 CoT response 更有效，因为 CoT 暴露了 claim；对 direct answer，尤其是短答案，它需要额外的 implicit answer claim reconstruction。否则 direct prompt 的 hallucination rate 可能被低估。

##### 9.5.4 CoT 同时改变了模型行为和可观测性

本实验中 CoT 的 hallucination rate 全面高于 direct，但这个结果不能直接解释为“CoT 使模型真实更容易幻觉”。CoT 至少同时改变了两件事：

1. **模型生成行为**：模型确实可能因为展开推理而生成额外错误，例如过度解释、错误中间推导或语义外推。
2. **错误可观测性**：CoT 把隐藏在 final answer 后面的视觉读取和推理步骤显式化，使 Judge 有更多 claim 可以检查。

因此，CoT 与 direct 的比较并不是完全等价的 prompt ablation。Direct answer 可能隐藏错误，CoT answer 可能暴露错误；response-level hallucination rate 上升既可能来自真实错误增加，也可能来自 claim exposure 增加。本报告通过 Claims / Resp. 和 Unsupported Claim Ratio 尝试区分两者，但仍无法完全分离生成效应和暴露效应。

更严格的后续设计可以引入同一 final answer 下的 post-hoc rationale 检查，或对 direct answer 构造隐式 claim，再与 CoT claim 做更公平的 claim-level 对齐。

##### 9.5.5 Human-as-Judge 样本量较小，结论应定位为诊断性

当前人工标注集包含 40 条样本，满足考核中 20–50 条小规模人工验证集的要求，也足以进行 failure pattern 分析。但从统计意义上看，40 条样本不足以支撑强显著性结论，尤其在按 dataset、model、prompt 和 fine label 拆分后，每个子组样本数更小。

例如 `ATT`、`INC`、`SO` 的 label-level alignment 能揭示重要错误线索，但不能被解读为稳定的总体性能估计。当前更合理的表述是：v2 在这个人工验证集上相对 baseline 更接近人类判断，并暴露出 MathVista attribute reading、implicit answer claim 和 reasoning-final answer consistency 等主要改进方向。

因此，Human-as-Judge 结果应作为 detector 设计的校准证据，而不是最终 benchmark 排名。若要得到更稳健结论，需要扩大人工标注规模，并增加多名标注者以计算 inter-annotator agreement。

##### 9.5.6 Reference answer 缺失会增加判断不确定性

MathVista 中部分样本的 `reference_answer` 为 `UNAVAILABLE`。这使 Judge 不能简单使用 reference answer 作为 contradiction evidence，只能依赖图像、题干、选项、数学规则和内部一致性。这样做可以避免把 answer-only mismatch 错判为 hallucination，但也会增加边界样本的不确定性。

在 reference 缺失时，Judge 很难区分三种情况：模型答案正确但缺少解释、模型答案错误但没有显式错误 claim、以及题目本身需要外部常识或不可见信息。`ann_0002`、`ann_0013` 和 `ann_0015` 都与这一问题有关。

因此，v2 的适用边界是：当 reference answer 完整、图像证据清晰、claim 可直接验证时，它更可靠；当 reference 缺失、答案很短、题目需要估计或外部常识时，它更容易出现 disagreement。

##### 9.5.7 Taxonomy 覆盖了主要错误，但仍有边界重叠

当前 taxonomy 使用 `OBJ/ATT/SPA` 表示 factual / visual hallucination，使用 `IR/CI/INC/SO` 表示 logical / semantic hallucination。这个体系能覆盖 POPE 与 MathVista 的主要错误，但实际案例中标签边界可能重叠。

例如图表数值读取错误可以标为 `ATT`，但如果错误数值导致后续推理错误，也可能表现为 `IR`；短答案语义外推可以标为 `SO`，但如果它源于视觉属性误读，也可能与 `ATT` 相关；reasoning 与 final answer 不一致可标为 `INC`，但如果不一致由题干语义绑定失败引起，也与 semantic parsing 有关。

因此，fine label 应被理解为主要错误来源，而不是互斥的唯一解释。报告中使用 primary label 便于统计，但 failure case 分析仍需要保留多因果解释。

##### 9.5.8 MLLM-as-a-Judge 本身可能存在系统性偏差

本实验使用多模态 LLM 作为自动 Judge，因此还需要考虑 MLLM-as-a-Judge 场景中的系统性偏差。即使 Judge 的输出被限制为 claim-level JSON schema，它仍可能受到模型自身偏好、回答长度、措辞风格、CoT 格式和图像-文本注意力分配的影响。

首先，Judge 可能出现 **modality neglect**：它在判断 response 是否 grounded 时，可能过度依赖文本回答的流畅性或自洽性，而没有充分复核图像证据。`ann_0010` 中 Judge 接受模型声称的动物计数，就是这种风险的体现。其次，Judge 可能存在 **verbosity / CoT bias**：较长的 CoT 回答提供了更多可检查 claim，也更容易暴露错误；但 Judge 也可能因为回答结构完整、解释看似合理而降低警惕，或者相反地，因为中间步骤较多而过度惩罚边界型推理。第三，Judge 可能存在 **model preference bias**：当被评估模型的表达风格与 Judge 偏好的格式相近时，Judge 可能更容易接受其解释；当回答较短、缺少推理或措辞不确定时，Judge 可能更容易判为 unsupported。

因此，Human-as-Judge 在本实验中不是附加验证，而是校准自动 Judge 偏差的必要环节。当前 40 条人工标注样本已经用于发现 v2 的 false positives 和 false negatives，但仍不足以完全刻画 MLLM judge 的系统性偏差。后续如果要把该方法用于更正式的 benchmark，应考虑增加多名人工标注者、扩大人工验证集、比较多个 Judge 模型、报告不同 Judge 之间的一致性，并分析 Judge 是否对 CoT 长度、模型来源、回答格式或任务类型存在稳定偏差。

##### 9.5.9 方法贡献与限制的平衡结论

综合来看，Zero-shot Judge v2 的贡献在于：它不训练额外模型，仍能通过 prompt engineering、claim-level schema 和 deterministic normalization 构建一个可解释、可复现、可与人类标注对齐的 hallucination detector。相较 baseline，v2 明显减少 false positives，更适合用于报告中的自动化评估。

但它的限制也很明确：它不能完全解决视觉感知错误，不能消除 hallucination 定义的主观边界，不能在短答案和 reference 缺失场景下稳定重构隐含 claim，也不能仅凭 40 条人工样本证明跨任务泛化性能；同时，作为 MLLM-as-a-Judge 方法，它还可能受到 modality neglect、verbosity / CoT bias 和 model preference bias 的影响。因而，本研究对 v2 的定位应是 **diagnostic zero-shot judge**：它适合发现趋势、比较模型、定位错误模式和辅助人工分析；若用于正式高风险评测，还需要更大规模人工标注、多 judge ensemble、任务专用规则或外部工具验证来增强可靠性。

XLRS-bench 的预期分析重点不是简单增加一个数据集，而是检验 unified claim-level judge 在遥感证据边界上的适用性：

- `OBJ`：建筑、道路、车辆、船只、水体、植被等目标或类别错误；
- `ATT`：数量、大小、纹理、地物属性或覆盖类型错误；
- `SPA`：空间布局、相邻关系、方向关系错误；
- `SO`：未经图像支持的土地用途、功能、意图或场景语义断言；
- `IR`：从遥感视觉证据到场景结论的无效推理。

因此，XLRS-bench 是当前方法的自然领域扩展：它保留同一 claim-level judge 协议，但把证据边界从常规 VQA / 数学图文推理推进到细粒度遥感视觉 grounding 与高层语义过度断言。
