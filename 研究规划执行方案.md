# 多模态大模型幻觉检测与评估：最终执行方案

## 0. 方案定位

本方案将原有研究蓝图收敛为一个 **主线清晰、工作量可控、结果可分析的考核型研究项目**。

核心思想是：

> 普通 Accuracy 不能充分评估多模态大模型的可靠性，因此需要结合幻觉分类、自动检测、human-as-judge 和 CoT 分析，判断模型回答是否真正 grounded in visual evidence。

最终项目聚焦两个核心问题：

1. **自动幻觉检测能否接近人类判断？**
2. **CoT 和模型类型如何影响多模态幻觉模式？**

---

## 1. 最终实验配置

采用如下最小但完整的实验配置：

```text
Datasets: POPE + MathVista
Models: 1 closed-source MLLM + 1 open-source MLLM
Prompt Settings: Direct Answer + Evidence-Grounded CoT
Detection Methods: Rule-based Detector + Zero-shot Judge + Checklist-based Judge
```

## 3. 简化后的幻觉分类体系

为了保证标注稳定、结果可解释，主统计只使用 3 类幻觉。

### 3.1 Object Hallucination

定义：

> 图像中不存在某个物体，但模型声称该物体存在。

主要对应 POPE。

示例：

```text
Image: 图中没有自行车。
Question: Is there a bicycle in the image?
Model: Yes, there is a bicycle.
Category: Object Hallucination
```

### 3.2 Visual Grounding Hallucination

定义：

> 模型对图像中的视觉证据产生错误理解，包括颜色、数量、空间关系、图表数值、OCR 文本、几何标注等错误。

覆盖原细分类中的 ATT、SPA、OCR/Text 等。

示例：

```text
Image: 图表中 2021 年数值为 30。
Question: What is the value in 2021?
Model: The value is 50.
Category: Visual Grounding Hallucination
```

### 3.3 Reasoning Hallucination

定义：

> 模型在推理或 CoT 中引入图像不支持的前提、错误逻辑、无依据假设或内部矛盾。

覆盖原细分类中的 IR、CI、INC、SO 等。

示例：

```text
Question: 求几何图中的角度。
Model reasoning: Since the triangle is equilateral, each angle is 60 degrees.
Problem: 图中没有等边三角形标记。
Category: Reasoning Hallucination
```

### 3.4 细分类的使用方式

OBJ、ATT、SPA、OCR/Text、IR、CI、INC、SO 不作为主结果表的统计类别，只在 failure case 中作为细粒度解释使用。

这样可以避免：

- 类别过多导致人工标注困难；
- 每类样本太少导致结果不稳定；
- LLM judge 在细分类上不一致。

## 4. Prompt 设计

最终只保留两种 prompt 设置。

### 4.1 Direct Answer Prompt

用于无 CoT 设置。

#### POPE

```text
Based only on the image, answer the question with "Yes" or "No".
Do not guess. Use only visible evidence from the image.

Question: {question}
```

#### MathVista

```text
Based only on the image, answer the question.
Provide only the final answer.
Do not use external knowledge or unsupported assumptions.

Question: {question}
```

### 4.2 Evidence-Grounded CoT Prompt

用于 CoT 设置。不要使用普通 “Think step by step”，而要要求模型先列出视觉证据。

```text
Based only on the image, answer the question.

Please provide:
1. Visual Evidence: list the visible facts needed to answer the question.
2. Reasoning: explain how the evidence supports the answer.
3. Final Answer: give the final answer.

If the image does not provide enough evidence, state that it is uncertain.
Do not introduce visual facts that are not visible in the image.

Question: {question}
```

这样设计的目的：

- 让 CoT 变成“证据约束推理”；
- 方便判断模型是否编造视觉证据；
- 方便 human annotator 和 LLM judge 检查 reasoning 是否 grounded。

## 5. 自动幻觉检测方法

最终组织成：两个 baseline + 一个 proposed method。

### 5.1 Baseline 1：Rule-based Detector

主要用于 POPE。

规则：

```text
If ground_truth = No and model_answer = Yes:
    hallucination = True
    hallucination_category = Object Hallucination
Else:
    hallucination = False
```

POPE 解释表：

| Ground Truth | Model Answer | Interpretation |
|---|---|---|
| Yes | Yes | Correct |
| Yes | No | Missed visual evidence |
| No | Yes | Object Hallucination |
| No | No | Correct rejection |

优点：

- 简单；
- 稳定；
- 可解释；
- 适合 yes/no object hallucination。

局限：

- 只能处理结构化 yes/no 样本；
- 不适合复杂 CoT 和开放式推理。

---

### 5.2 Baseline 2：Zero-shot LLM Judge

用于 MathVista 和复杂输出。

输入：

```text
Question
Reference Answer
Model Response
Visual Evidence, if provided by model
Reasoning, if provided by model
Final Answer
```

输出 JSON：

```json
{
  "answer_correct": true,
  "is_hallucination": true,
  "hallucination_category": "Reasoning Hallucination",
  "unsupported_visual_claim": true,
  "confidence": "high",
  "explanation": "The model assumes that two lines are parallel, but the image does not show a parallel marker."
}
```

Judge prompt 中必须强调：

- 答案错误不自动等于幻觉；
- 答案正确也可能存在幻觉；
- 只根据图像、问题、参考答案和模型输出判断；
- 如果 reasoning 中出现图像不支持的视觉事实，应标记为 hallucination；
- 如果证据不足，应标记为 unclear 或 low confidence。

---

### 5.3 Proposed Method：Few-shot Checklist-based Judge

这是最终方案中的主要方法亮点。

核心思想：

> 不直接问 judge “是否有幻觉”，而是先检查关键视觉事实和推理依据，再判断幻觉类别。

方法结构：

```text
Step 1: 检查模型是否回答正确。
Step 2: 检查模型是否提出 unsupported visual claim。
Step 3: 判断是否存在 hallucination。
Step 4: 在 3 个主类别中选择 hallucination category。
Step 5: 输出简短 explanation。
```

Few-shot examples 应覆盖：

- Object Hallucination；
- Visual Grounding Hallucination；
- Reasoning Hallucination；
- No Hallucination；
- Correct Answer with Wrong Evidence；
- Ambiguous / Unclear case。

与 zero-shot judge 的区别：

```text
Zero-shot judge: 直接判断整体是否幻觉。
Checklist-based judge: 先检查视觉证据和推理依据，再判断幻觉。
```

该方法吸收三篇文献思想：

- 来自 Seeing Clearly without Training：答案正确也可能有幻觉，答案错误不必然是幻觉；
- 来自 PEARL：区分 perception error 和 reasoning error，检查视觉证据；
- 来自 Omni-I2C：使用 checklist / atomic facts 降低 judge 随机性。

## 6. Perception Checklist 的使用范围

Perception Checklist 是方案亮点，但不做全量，以控制工作量。

只用于三处：

1. **MathVista human annotation 子集**；
2. **CoT failure cases**；
3. **Few-shot checklist-based judge 的 prompt 设计**。

### 6.1 Checklist 示例

#### 几何题

```text
Main Question: What is the missing angle?
Perception Checklist:
1. Which angle is labeled 40 degrees?
2. Is there a right-angle marker?
3. Are any two lines marked as parallel?
4. Which triangle contains the unknown angle?
```

#### 图表题

```text
Main Question: Which year has the largest increase?
Perception Checklist:
1. What are the x-axis labels?
2. What is the value in 2020?
3. What is the value in 2021?
4. Which bar is the highest?
```

### 6.2 Checklist 目的

用于判断：

- 模型是否真的看到了关键视觉事实；
- CoT 中哪些视觉证据是被图像支持的；
- 模型是否出现 “answer correct but evidence wrong”。

Human-as-Judge 是考核重点，必须作为核心实验之一。

### 7.1 样本数量

固定标注 **40 条样本**。

### 7.2 分层抽样

- 两种 prompt；
- 正确和错误样本；

### 7.3 人工标注展示内容

每条样本给标注者展示：

```text
Image
Question
Reference Answer
Model Response
Prompt Type

```

如果是 Evidence-Grounded CoT，还展示：

```text
Visual Evidence
Reasoning
Final Answer
```

### 7.4 人工标注字段

为了保证标注稳定，只保留 6 个字段：

```text
answer_correct: Yes / No / Unclear
is_hallucination: Yes / No / Unclear
hallucination_category: Object / Visual Grounding / Reasoning / None / Unclear
unsupported_visual_claim: Yes / No / Unclear
confidence: High / Medium / Low
short_explanation: one or two sentences
```

不要让人工标注者同时标注 OBJ、ATT、SPA、IR、CI、INC、SO 等细类，避免复杂度过高。


## 8. 主指标设计

最终主指标只保留 5 个。

### 8.1 Task Accuracy

衡量最终答案正确率。

```text
Task Accuracy = correct answers / total samples
```

作用：提供传统任务性能基线。

---

### 8.2 Hallucination Rate

衡量模型输出中有多少被判定为幻觉。

```text
Hallucination Rate = hallucinated responses / total responses
```

POPE 上可特别报告：

```text
Object Hallucination Rate = false positives on negative object questions / total negative object questions
```

---

### 8.3 Hallucination Type Distribution

统计三类幻觉比例：

```text
Object Hallucination
Visual Grounding Hallucination
Reasoning Hallucination
```

作用：分析不同模型和 prompt 的错误模式。

---

### 8.4 Detector-Human F1

以人工标注为 gold label，评估自动检测方法。

```text
Positive class = hallucination
```

报告：

- Precision；
- Recall；
- F1。

重点比较：

```text
Rule-based Detector vs Zero-shot Judge vs Checklist-based Judge
```

---

### 8.5 Grounded Accuracy

这是本项目最有研究价值的指标。

定义：

```text
Grounded Accuracy = 答案正确，且没有 unsupported visual claim 的样本比例
```

对于 CoT 样本，要求：

```text
Final Answer correct
AND Visual Evidence supported by image
AND Reasoning does not introduce unsupported assumptions
```

意义：

> 普通 Accuracy 只看答案是否正确，而 Grounded Accuracy 进一步检查答案是否建立在真实视觉证据上。

这个指标可以突出：

- 普通 Accuracy 会高估 MLLM 可靠性；
- 有些模型答案正确但推理依据错误；
- CoT 可能提升答案正确率，但不一定提升 grounded reliability。

## 10. Failure Case 分析

至少展示 2 个，建议准备 4 个。

### Case 1：POPE Object Hallucination

模型说图中有不存在的物体。

分析重点：

```text
错误是否来自 yes-bias？
模型是否被问题中的 object name 诱导？
rule-based detector 和 human judge 是否一致？
```

---

### Case 2：MathVista Visual Grounding Error

模型读错图表、数字、几何标注、数量或 OCR 文本。

分析重点：

```text
错误是否发生在 perception 阶段？
后续推理是否建立在错误视觉事实上？
Checklist 是否能定位错误来源？
```

---

### Case 3：CoT Reasoning Hallucination

模型在 CoT 中编造图像不支持的前提。

分析重点：

```text
CoT 是否引入 unsupported assumption？
错误是否由长推理放大？
Checklist-based judge 是否能识别该问题？
```

---

### Case 4：Correct Answer with Wrong Evidence

最终答案正确，但推理依据错误。

分析重点：

```text
为什么普通 Accuracy 无法发现该问题？
Grounded Accuracy 如何暴露该问题？
这个案例说明 MLLM 的可靠性风险是什么？
```
